# 维度三：多玩法共用的积分/货币体系

## 1. 棋牌平台货币体系的共性架构

### 双层货币模型
棋牌平台普遍采用"硬通货+软通货"的双层货币结构：

| 货币类型 | 获取方式 | 消耗场景 | 示例 |
|---------|---------|---------|------|
| 硬通货（充值货币） | 人民币充值 | 高级道具、赛事报名、VIP特权 | 金币/钻石 |
| 软通货（活跃货币） | 日常任务、登录奖励、比赛奖励 | 普通场次入场、低级道具 | 银币/积分 |

来源: [虚拟币体系搭建](http://www.woshipm.com/operate/1746703.html)

### 多玩法间的购买力平衡
核心难题：同一个"金币"在斗地主场和麻将场的购买力如何一致？

**方案一：统一入场费倍率**
- 所有玩法按"初级场/中级场/高级场"分档
- 同档位入场费完全一致（如初级场统一100金币）
- 不同玩法的"一局时长"差异通过倍率补偿

**方案二：玩法独立积分+兑换中心**
- 每种玩法有独立的"经验分"或"段位分"
- 统一货币（金币）只用于入场和购买
- 通过兑换中心实现不同玩法积分的互通

**方案三：赛事积分独立体系**
- JJ竞技采用此模式：比赛奖励为"虚拟物品"，可在奖品商城兑换实物
- 游戏币和赛事积分完全分离
- 避免不同玩法间的经济体膨胀互相传导

## 2. 微乐的货币设计

### 轻货币化策略
微乐作为腾讯系产品，采用较轻的货币化策略：
- **免费为主**：核心玩法免费，不设入场费门槛
- **道具付费**：表情、装饰、特效等社交类道具收费
- **广告变现**：游戏内广告是重要收入来源
- **熟人房模式**：房卡/钻石创建房间是核心付费点

### 积分设计
- 胜负积分：影响匹配对手等级
- 活跃积分：登录、对局获取，兑换小奖品
- 无重度充值压力，符合腾讯"社交+轻游戏"定位

## 3. JJ竞技的赛事经济体

### 竞技积分体系
JJ比赛的货币体系围绕"赛事"设计：
- **比赛币**：用于报名各种比赛
- **赛事奖品**：比赛获胜赢得虚拟物品（如话费卡、实物奖品兑换券）
- **积分排名**：长期积分决定选手等级和参赛资格

来源: [JJ比赛官网](https://www.jj.cn/)

### 经济体闭环
```
充值/任务 → 获得比赛币 → 报名比赛 → 赢得奖品
                ↑                      ↓
              复投 ←──── 奖品兑换/消耗
```

## 4. 途游的混合模式

### 广告+内购双引擎
- 免费对局为主，广告穿插变现
- 高端赛事设置报名门槛
- 赛事奖品（家电、红包）驱动付费转化

## 5. 多玩法货币体系设计框架

### 核心原则
1. **通胀隔离**：不同玩法的经济体不应互相传导通胀
2. **购买力锚定**：同一货币在不同玩法场景的"每分钟消耗"应大致一致
3. **沉没成本分散**：避免用户只在一个玩法沉淀资产，鼓励跨玩法流转
4. **合规红线**：虚拟货币不可逆向兑换为人民币（合规要求）

### 推荐架构
```
┌──────────────────────────────────────┐
│           统一钱包层                  │
│    充值货币(钻石) + 活跃货币(银币)    │
├──────────┬──────────┬────────────────┤
│  玩法A   │  玩法B   │   玩法C        │
│ 段位分A  │ 段位分B  │  段位分C       │
│ 入场费A  │ 入场费B  │  入场费C       │
├──────────┴──────────┴────────────────┤
│           赛事积分层                  │
│    赛事积分 → 奖品商城 → 实物兑换    │
└──────────────────────────────────────┘
```
