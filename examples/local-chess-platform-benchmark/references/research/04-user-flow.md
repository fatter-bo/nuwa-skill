# 维度四：用户流转设计

## 1. 玩法间流转的核心挑战

棋牌平台承载几十种地方玩法，用户流转设计的核心问题：
- 用户进入平台后只玩家乡玩法，不尝试其他→玩法利用率低
- 新上线玩法无人问津→研发投入浪费
- 用户在单一玩法上倦怠后流失→平台留存受损

## 2. 微乐的流转设计

### 地理位置驱动的冷启动
- 用户首次进入，根据定位推荐家乡玩法→零决策成本
- 大厅页面展示"附近热门"和"全国热门"两个维度
- 用户可以"添加玩法"主动发现新玩法

### 社交关系链驱动流转
微乐最强的流转引擎是微信关系链：
- **好友在玩**：显示微信好友正在玩的玩法，制造好奇心
- **邀请入房**：好友发来的不同玩法房间邀请，天然的跨玩法引导
- **排行榜**：跨玩法的好友排行，激发竞争心理
- **分享机制**：对局结果分享到微信群，吸引群内其他人尝试

### 推荐算法
- 基于用户已玩玩法推荐相似玩法（麻将→其他地区麻将；斗地主→掼蛋）
- 基于同城/同省玩家行为推荐热门玩法
- 新玩法上线时对目标地区用户精准推送

## 3. JJ竞技的赛事驱动流转

### 跨玩法赛事
- 综合型赛事：一场比赛包含多种游戏项目，迫使选手接触不同玩法
- 赛事奖品吸引：高价值赛事奖品引导用户尝试不同玩法的比赛
- 限时赛事：特定时段开放特定玩法的比赛，制造稀缺感

### 新玩法曝光机制
- 大厅首页Banner位轮播新玩法赛事
- "新手保护赛"：新玩法设置低门槛入门比赛
- 赛事日历：提前公布各玩法赛程，培养用户预期

## 4. 途游的内容驱动流转

### 活动运营
- 节日活动覆盖多个玩法，引导用户跨玩法参与
- 每日任务设计跨玩法目标（如"今日在3种不同玩法中各赢一局"）
- 赛季pass：赛季任务覆盖全品类，鼓励广泛体验

### 内容更新节奏
- 定期推出限时玩法活动
- 通过直播、短视频展示新玩法亮点
- 社区UGC内容（攻略、精彩对局）引流新玩法

## 5. 用户流转设计框架

### 流转漏斗模型
```
[曝光] → [兴趣] → [尝试] → [留存] → [精通] → [传播]
  ↑         ↑         ↑        ↑         ↑         ↑
Banner   好友在玩  新手引导  段位成长  赛事体系  社交分享
热点推荐  赛事预告  保护机制  任务奖励  排行榜   对局回放
```

### 关键流转节点设计

| 节点 | 设计要点 | 避免陷阱 |
|------|---------|---------|
| 首次曝光 | 不超过3个推荐，避免选择困难 | 推荐太多导致决策瘫痪 |
| 尝试转化 | 提供AI对手试玩，无需等待匹配 | 直接扔进真人场被虐退 |
| 新手保护 | 前N局匹配同等新手，保证胜率 | 保护期太长导致上分后落差大 |
| 跨玩法引导 | 利用已有技能的迁移（麻将→麻将变体） | 推荐完全不相关的玩法 |
| 回流召回 | 沉默用户推送其朋友的近期动态 | 频繁推送造成骚扰 |

### 社交裂变的三级火箭
```
第一级：熟人房对局（最强场景驱动力）
第二级：对局结果分享（好奇心驱动）
第三级：排行榜竞争（持续动力）
```
