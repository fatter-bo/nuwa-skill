# 棋牌游戏数值体系

> 棋牌游戏的数值设计决定三件事：对局是否公平、经济是否可持续、玩家是否留存。
> 概率是地基，经济系统是上层建筑，防作弊是安全围栏。三者缺一不可。

## Skill 规则

- 所有概率计算必须给出精确数学推导过程，不接受"大约"或"经验值"
- 数值方案必须包含可验证的蒙特卡洛模拟建议（样本量、置信区间）
- 经济系统设计必须同时给出产出渠道和消耗池，并说明平衡逻辑
- 盲注/倍率结构必须附带不同场景下的EV分析
- 涉及RNG时必须指定算法级别（CSPRNG而非普通PRNG）
- 匹配系统推荐必须说明选择ELO/Glicko/Glicko-2的理由及各自局限

### 1.1 扑克牌组合数学

**52张标准扑克的5张手牌**：总组合数 C(52,5) = 2,598,960

| 同花顺 | 36 | 0.00139% | 1:72,193 |
| 四条 | 624 | 0.0240% | 1:4,165 |
| 葫芦 | 3,744 | 0.1441% | 1:694 |
| 同花 | 5,108 | 0.1965% | 1:509 |
| 顺子 | 10,200 | 0.3925% | 1:255 |
| 三条 | 54,912 | 2.1128% | 1:47 |
| 两对 | 123,552 | 4.7539% | 1:21 |
| 一对 | 1,098,240 | 42.2569% | 1:1.37 |

### 1.2 麻将组合数学

标准麻将136张牌（34种各4张），起手13张。胡牌基本结构：4组面子(顺子/刻子) + 1对雀头。

**关键概率参考值**：

| 牌型 | 近似概率 | 说明 |
|------|---------|------|
| 清一色 | ~1/32 | 单花色组成的完整手牌 |
| 九莲宝灯 | ~0.0113% | 1112345678999+任意同花色 |
| 大四喜 | ~1/57,856 | 四组风牌刻子 |
| 平和 | ~15-25% | 全顺子+非役牌雀头（日麻） |

**向听数(Shanten Number)**：从当前手牌到胡牌所需的最少换牌数。向听数计算是麻将AI的核心算法，复杂度来源于组合爆炸——需要遍历所有可能的面子拆解方式。

### 1.3 期望值(EV)计算框架

**通用EV公式**：

```
EV = P(win) × Amount_Won - P(lose) × Amount_Lost
```

**德州扑克Pot Odds**：

```
所需胜率 = 跟注金额 / (当前底池 + 跟注金额)
若实际胜率 > 所需胜率，则跟注为 +EV
```

**Implied Odds扩展**：考虑后续街可能赢得的额外筹码，适用于听牌场景。击中听牌后的潜在收益越大，当前不利的Pot Odds也可能是+EV。

### 1.4 蒙特卡洛模拟验证

理论概率需要模拟验证，特别是在条件概率复杂的场景下：

- 样本量要求：≥100万次模拟以获得稳定结果
- 置信区间：95%置信水平下，误差 ≈ 1.96 × sqrt(p(1-p)/n)
- 适用场景：已知部分公共牌时的胜率计算、多人对局胜率、特定手牌范围的EV计算
- 实现验证：Java/Python模拟结果与理论概率的吻合度通常在0.01%以内

## 二、各玩法数值体系

### 2.1 德州扑克

#### 盲注结构设计

| 参数 | SNG | MTT | Cash Game |
|------|-----|-----|-----------|
| 起始筹码/大盲 | 50-100BB | 100-300BB | 固定(通常100BB买入) |
| 盲注递增间隔 | 8-12min | 15-20min | 不递增 |
| Turbo变体 | 5-8min | 8-12min | N/A |
| Hyper-Turbo | 3-5min | 5-8min | N/A |
| Ante引入时机 | 第4-5级 | 第5-8级 | 可选 |

**盲注递增倍率**：相邻两级盲注的比值通常在1.25-1.67之间。过快(>2x)导致运气成分过大，过慢(<1.2x)导致赛事时间过长。

#### ICM模型

ICM (Independent Chip Model) 将锦标赛筹码转换为美元期望值。核心原理：**筹码的边际价值递减**——从0到10000筹码的价值远大于从90000到100000。

关键应用场景：
- **泡沫期决策**：Bubble Factor > 1，意味着输掉筹码的损失大于赢得筹码的收益
- **Final Table协议**：基于ICM计算每位玩家的公平分配
- **推注/弃牌范围调整**：ICM压力下收紧推注范围

#### GTO平衡

GTO(Game Theory Optimal)策略在数值设计中的应用：
- 混合策略频率（如70%价值下注 / 30%诈唬）确保不可被剥削
- 下注尺寸与价值/诈唬比例的数学关系：bet/(bet+pot) = 诈唬频率
- Solver工具（PioSOLVER、GTO Wizard）用于验证理论平衡

### 2.2 麻将番型倍率

#### 日本立直麻将计分

基于翻(Han)+符(Fu)双维度计分：

| 翻数 | 等级 | 基本点 | 闲家自摸(各付) | 庄家自摸(各付) |
|------|------|--------|---------------|---------------|
| 1-4翻 | 按翻符计算 | 公式算 | 公式算 | 公式算 |
| 5翻(或4翻40+符) | 满贯(Mangan) | 2,000 | 2,000/4,000 | 4,000 |
| 6-7翻 | 跳满(Haneman) | 3,000 | 3,000/6,000 | 6,000 |
| 8-10翻 | 倍满(Baiman) | 4,000 | 4,000/8,000 | 8,000 |
| 11-12翻 | 三倍满(Sanbaiman) | 6,000 | 6,000/12,000 | 12,000 |
| 13翻+ | 役满(Yakuman) | 8,000 | 8,000/16,000 | 16,000 |

每+1翻约翻倍，直至满贯上限后阶梯递增。Dora(宝牌)机制每张+1翻，增加随机性和刺激感。

#### 中国地方麻将差异

| 地方规则 | 核心特色 | 对胡牌概率的影响 |
|---------|---------|----------------|
| 四川血战 | 缺一门，血战到底 | 缺门限制降低有效牌种，但血战到底增加多次胡牌机会 |
| 广东推倒胡 | 无番数限制，任意胡 | 胡牌门槛最低，对局节奏最快 |
| 武汉麻将 | 赖子(万能牌) | 赖子大幅提高胡牌概率，需要调高番型倍率补偿 |
| 国标麻将(MCR) | 最低8番，81种番型 | 门槛高，平均对局时间长，大牌概率更受关注 |

### 2.3 斗地主倍率体系

**完整得分公式**：

```
最终得分 = 基础分 × 叫分倍率 × 2^(炸弹数+火箭数) × 春天倍率
```

| 组件 | 倍率 | 说明 |
|------|------|------|
| 叫分 | 1/2/3 | 玩家竞叫，最高者为地主 |
| 炸弹(4张相同) | ×2 / 个 | 累乘，2个炸弹=×4 |
| 火箭(双王) | ×2 | 等效一个炸弹 |
| 春天 | ×2 | 地主出完牌时农民一张未出 |
| 反春天 | ×2 | 农民先出完且地主仅出过一手 |

**地主角色倍率**：地主赢/输 = 2倍基础得分（因为对抗两位农民）。农民各承担一半。

**EV平衡分析**：叫3分时若出现2个炸弹+春天，倍率=3×4×2=24。需要评估极端倍率出现的概率，确保不会导致经济系统剧烈波动。

### 2.4 21点与百家乐

**21点(Blackjack)**：
- 完美基本策略庄家优势：~0.5%
- 无策略玩家庄家优势：~2-5%
- 6副牌标准规则，算牌可将优势反转至玩家(+0.5%~+1.5%)

**百家乐(Baccarat)**：

| 投注 | 赢率 | 庄家优势 |
|------|------|---------|
| 庄 | 45.8% | 1.06% |
| 闲 | 44.6% | 1.24% |
| 和 | 9.6% | 14.36% |

百家乐的数学结构使得算牌几乎无效（与21点不同），因为发牌规则是预定义的，玩家无决策空间。

## 三、欢乐豆经济系统

### 3.1 水龙头-排水口(Faucet-Drain)模型

棋牌游戏经济的核心平衡：**产出速率 ≈ 消耗速率**。

```
┌─────────────────────────────────────────────────┐
│                  货币产出(水龙头)                   │
│                                                   │
│  每日登录  对局奖励  成就任务  好友赠送  广告奖励  充值  │
│     ↓        ↓        ↓        ↓        ↓      ↓   │
│  ═══════════════ 玩家钱包 ═══════════════════════  │
│     ↓        ↓        ↓        ↓        ↓      ↓   │
│  台费抽水  房间门槛  道具购买  锦标赛费  VIP订阅  过期  │
│                                                   │
│                  货币消耗(排水口)                   │
└─────────────────────────────────────────────────┘
```

### 3.2 产出渠道设计

| 渠道 | 日产出量(建议) | 占比 | 说明 |
|------|--------------|------|------|
| 每日登录 | 5,000-10,000 | 15-20% | 基础产出，保障活跃 |
| 对局奖励(赢局) | 按对局级别 | 30-40% | 零和转移为主，系统补贴为辅 |
| 任务/成就 | 3,000-8,000 | 10-15% | 引导行为，有上限 |
| 广告奖励 | 2,000-5,000 | 5-10% | 变现联动，每日次数限制 |
| 好友赠送 | 1,000-3,000 | 5% | 社交拉动，防刷限制 |
| 破产救济 | 10,000-30,000 | N/A | 零货币时触发，每日限次 |
| 充值购买 | 无上限 | 20-30% | 核心变现，阶梯定价 |

### 3.3 消耗池设计

| 消耗方式 | 消耗比例 | 说明 |
|---------|---------|------|
| 台费抽水 | 3-5%/局 | 从赢家收取，核心Sink |
| 高级房门槛 | 梯度递增 | 初级1K，中级10K，高级100K |
| 道具/表情 | 单次500-5,000 | 社交消耗，非必要 |
| 锦标赛报名 | 按级别 | 奖金池分配后剩余为Sink |
| 皮肤/装饰 | 10,000-100,000 | 高价值装饰品 |

### 3.4 通胀控制模型

**监控指标**：
- **M2总量**：服务器所有账户货币总和
- **月增长率**：M2的月环比增长，目标 < 5%
- **基尼系数**：货币分布均匀度，目标 < 0.6
- **破产率**：零货币账户比例，目标 < 15%

**动态调控机制**：

| 条件 | 操作 |
|------|------|
| M2月增长 > 8% | 提高台费抽水率0.5%，减少登录奖励10% |
| 基尼系数 > 0.7 | 增加破产救济金额，降低高级场门槛 |
| 破产率 > 20% | 增加免费产出渠道，投放限时活动 |
| 破产率 < 5% | 经济过于宽松，可考虑增加消耗项 |

### 3.5 防刷机制

| 风险行为 | 检测方式 | 处置 |
|---------|---------|------|
| 同设备多开 | 设备指纹关联 | 限制同设备同时在线账户数 |
| 协议刷分 | 固定对手+快速输赢模式 | 异常对局不计入收益 |
| 脚本挂机 | 操作间隔统计分析 | 触发人机验证 |
| 大额异常转移 | 短时间大额净转移 | 冻结+人工审核 |
| 新账户薅羊毛 | 注册时间+行为模式 | 新账户限制赠送/交易 |

## 四、匹配与房间系统

### 4.1 Rating算法选型

| 算法 | 适用场景 | 优势 | 局限 |
|------|---------|------|------|
| **ELO** | 1v1对局为主、用户基数大 | 简单直观、计算快 | 不考虑不确定性，长期不活跃玩家分数失真 |
| **Glicko-2** | 玩家活跃度差异大、需精确匹配 | 含不确定性(RD)+波动性(σ) | 计算复杂度高，存在Volatility Farming漏洞 |
| **Elo-MMR** | 多人锦标赛排名 | 处理N人同场竞技 | 实现复杂度最高 |

**推荐方案**：休闲棋牌用ELO即可（实现简单、用户基数大时精度够用）。竞技向棋牌用Glicko-2（需要处理活跃度差异）。锦标赛排名用Elo-MMR。

#### ELO核心公式

```
预期胜率: E_A = 1 / (1 + 10^((R_B - R_A) / 400))
更新公式: R'_A = R_A + K × (S_A - E_A)

K值选择:
  新玩家(前30局): K = 32  (快速定位)
  稳定玩家: K = 16  (缓慢微调)
  高段位玩家: K = 10  (更加稳定)
```

#### Glicko-2扩展参数

```
Rating (μ): 技能评分，初始1500
Rating Deviation (φ): 不确定性，初始350，活跃后降低
Volatility (σ): 表现波动性，初始0.06
置信区间: 约95%真实实力在 [μ - 2φ, μ + 2φ]
```

### 4.2 等级场分层设计

```
VIP场:    1,000,000+  盲注 5,000/10,000   ← 高端玩家
高级场:   100,000+    盲注 500/1,000       ← 中高端
中级场:   10,000+     盲注 50/100          ← 大众玩家
初级场:   1,000+      盲注 5/10            ← 新手/休闲
新手场:   0+          免费                  ← 纯新手保护区
```

**跨层控制**：
- 每层设置携带下限和上限（防止大鱼屠幼鱼）
- 升层后至少完成5局才能降回（防快速切层套利）
- VIP场可设置邀请制或信誉分门槛

### 4.3 匹配队列优化

**渐进放宽策略**：

```
0-10s:  Rating差 ≤ ±50   (严格匹配)
10-20s: Rating差 ≤ ±100  (适度放宽)
20-30s: Rating差 ≤ ±200  (较大放宽)
30s+:   任意Rating / Bot填充
```

目标：中位数等待时间 < 15秒，P95 < 30秒。

### 4.4 Bot填充策略

| 场景 | Bot比例上限 | Bot难度 | 透明度 |
|------|-----------|---------|--------|
| 新手场 | 100% | 低(随机+基础规则) | 不标注 |
| 低谷时段 | 50% | 匹配Rating对应级别 | 可选标注 |
| 高级场 | 0% | N/A | 全真人 |
| 锦标赛 | 0% | N/A | 全真人 |

Bot行为拟人化要点：模拟人类思考时间分布（非固定延迟）、引入可控犯错率、模拟操作节奏变化。

## 五、防作弊数值校验

### 5.1 RNG公平性保证

**必须使用CSPRNG（密码学安全伪随机数生成器）**：

| 算法 | 适用 | 不可用 |
|------|------|--------|
| AES-256-CTR | 生产环境 | |
| ChaCha20 | 生产环境 | |
| Fortuna | 生产环境 | |
| Mersenne Twister | | 不安全，可被预测 |
| Math.random() | | 不安全，非密码学级别 |

**Fisher-Yates洗牌算法**（标准实现）：

```
for i from n-1 down to 1:
    j = CSPRNG_random_int(0, i)  // 包含两端
    swap(deck[i], deck[j])
```

时间O(n)，空间O(1)。关键：随机源必须是CSPRNG。常见错误包括Sattolo变体（只产生环排列）和模偏差（random() % n不均匀）。

### 5.2 可证公平(Provably Fair)方案

**Commit-Reveal方案**：

```
发牌前:
  server_seed = CSPRNG生成
  client_seed = 玩家提供(或自动生成)
  nonce = 递增计数器
  deck_hash = SHA-256(server_seed + client_seed + nonce)
  → 将deck_hash发送给玩家

对局中:
  牌序 = Fisher-Yates(seed = HMAC(server_seed, client_seed + nonce))
  → 正常发牌和对局

对局后:
  公开server_seed → 玩家可重算deck_hash验证一致性
  → 可验证牌序在发牌前已确定，运营方无法篡改
```

### 5.3 异常行为检测维度

| 检测维度 | 指标 | 阈值建议 | 处置 |
|---------|------|---------|------|
| **共谋** | 固定对手频率、让利行为模式 | 同对手>20局/日且净转移>3σ | 标记审查 |
| **刷分** | 胜率偏离、收益曲线斜率 | 胜率>3σ或收益增速>5σ | 冻结+回滚 |

## 核心方法论框架

### 框架一：组合概率精确计算法

所有牌型概率必须基于组合数学精确推导，不接受经验估计。计算路径：确定总样本空间 → 分解有利事件 → 用乘法原理/加法原理计数 → 蒙特卡洛模拟验证（≥100万次）。理论值与模拟值偏差>0.1%时需检查推导过程。

来源：[Poker probability - Wikipedia](https://en.wikipedia.org/wiki/Poker_probability)、[McGill MATH 323](https://www.math.mcgill.ca/dstephens/323/knitr/knit-01-Combinatorics.pdf)

### 框架二：水龙头-排水口经济平衡模型

虚拟经济系统的稳定性取决于产出(Faucet)与消耗(Drain)的平衡。棋牌游戏的核心Sink是台费抽水（3-5%），核心Faucet是对局奖励+每日登录。通过监控M2总量、基尼系数、破产率三个指标实时调控，月通胀率目标<5%。

来源：[1kxnetwork Sinks & Faucets](https://medium.com/1kxnetwork/sinks-faucets-lessons-on-designing-effective-virtual-game-economies-c8daf6b88d05)、[Lost Garden Value Chains](https://lostgarden.com/2021/12/12/value-chains/)、[Machinations Inflation](https://machinations.io/articles/what-is-game-economy-inflation-how-to-foresee-it-and-how-to-overcome-it-in-your-game-design)

### 框架三：ICM边际价值递减模型

锦标赛中筹码的真实价值不等于面值。ICM模型通过各玩家筹码占比计算每人获得各名次的概率，再乘以奖金结构得到美元期望值。关键推论：泡沫期的Bubble Factor>1，决策应比筹码模型(cEV)更保守。

来源：[GGPoker ICM Guide](https://ggpoker.com/blog/intermediate-strategy/understanding-icm-an-essential-component-of-poker-strategy/)、[GTO Wizard Bubble Factor](https://blog.gtowizard.com/what-is-the-bubble-factor-in-poker-tournaments/)

### 框架四：Glicko-2不确定性匹配模型

在ELO基础上增加Rating Deviation(不确定性)和Volatility(波动性)两个维度。长期不活跃玩家的RD自动增大，重新活跃后匹配权重调整更激进以快速重新定位。适用于棋牌游戏中玩家活跃度差异大的场景。

来源：[ELO vs Glicko Poker Ranking](https://pokergamedevelopers.com/elo-vs-glicko-poker-ranking-system/)、[Glicko-2 Enhancement Paper](https://www.cimachinelearning.com/assets/article/vol6-iss1/glicko-2-algorithm.pdf)

通过密码学承诺方案证明牌序在发牌前已确定且不可篡改。发牌前公开牌序哈希(Commit)，对局后公开种子(Reveal)，任何人可验证一致性。这是比第三方认证更透明的信任模型，适用于需要高度信任的竞技场景。

## 决策启发式

1. **"概率先行"原则**：设计任何倍率/赔率体系前，先完成精确的概率计算。直觉估计的"大约1%"和精确计算的"0.0240%"之间可能差两个数量级，这会导致经济系统完全失衡。

2. **"3-5%抽水率"原则**：棋牌游戏台费抽水率的健康区间是3-5%。低于3%不足以对抗通胀，高于5%玩家感知到"被割"会加速流失。精确值需通过A/B测试确定。

3. **"倍率封顶"原则**：任何包含连续翻倍机制的玩法（如斗地主炸弹），必须设置单局最大倍率上限。否则极端情况（4炸弹+春天=64倍）会导致单局货币波动超出经济系统承受能力。

4. **"破产不流失"原则**：玩家破产是变现机会，但也是流失节点。破产救济金必须足够玩5-10局初级场——让玩家有机会回血或产生付费意愿，而不是直接关掉游戏。

5. **"K值分段"原则**：ELO的K值不能一刀切。新玩家K=32（快速定位真实水平），稳定玩家K=16（避免波动），高段位K=10（维护排名严肃性）。错误的K值会让匹配体验恶化。

6. **"Bot拟人≠Bot最优"原则**：Bot的目的是填充匹配队列、提供合理对手，而不是展示AI实力。Bot必须引入可控的犯错率和模拟人类思考时间，否则玩家会感知到"不像人"并产生不信任。

7. **"CSPRNG不可妥协"原则**：洗牌的随机源必须是AES-256-CTR、ChaCha20等密码学安全算法。Mersenne Twister、Math.random()等非CSPRNG在理论上可被预测，这在涉及真实价值的场景中是不可接受的安全漏洞。

8. **"D7累计ARPU"原则**：在棋牌游戏中评估数值调整效果时，D7累计ARPU是最佳单一指标。它同时反映留存（用户是否还在）和变现（用户是否付费），比单看留存或单看ARPU更全面。

9. **"新手保护平滑退出"原则**：新手保护期结束不能断崖式切换。保护期(30局)→过渡期(30局)→正常期的渐进设计，比直接取消保护的D30留存通常高5-10个百分点。

10. **"模拟验证理论"原则**：任何概率计算结果在投入生产前，都必须经过≥100万次蒙特卡洛模拟验证。理论值与模拟值偏差超过0.1%时，优先怀疑理论推导而非模拟代码。

## 诚实边界

1. **概率计算有前提假设**：本文所有扑克/麻将概率基于完全随机洗牌和均匀分配。实际物理洗牌可能存在不充分混洗的偏差，线上洗牌依赖RNG质量——概率精确性的上限是RNG的质量。

2. **经济模型是简化**：水龙头-排水口模型是一阶近似。真实的棋牌经济还受到玩家心理账户效应、损失厌恶、社交攀比等行为经济学因素影响，这些无法纯数学建模。

3. **ICM模型假设独立性**：ICM假设每手牌独立且每位玩家技术水平相同。实际中技术差异显著，ICM提供的是数学下界，不是最优策略。

4. **Glicko-2存在已知漏洞**：Volatility Farming攻击可使恶意玩家人为膨胀数百分Rating。部署时需要额外的反滥用检测层。

5. **防作弊是军备竞赛**：检测系统和作弊工具在持续对抗演进。2025年的RTA检测方案可能在2026年被新工具突破。没有一劳永逸的解决方案。

6. **A/B测试在棋牌游戏中特别困难**：ARPU的极度偏斜分布、匹配系统的网络效应、数值调整的长尾效应，都使得标准A/B测试方法论需要显著修正。小团队可能缺乏足够流量获得统计显著性。

7. **各地方麻将规则变体极多**：本文覆盖的是主流规则框架。实际中，仅中国大陆就有数十种地方变体，每种的番型定义和概率不同。具体项目需要针对目标规则单独计算。
