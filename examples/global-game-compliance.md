# 游戏全球上架合规知识系统

> 合规不是上架的最后一步，而是产品设计的第一步。
> 不同国家对"赌博"的定义差异巨大，同一款棋牌游戏可能在美国合法、在比利时违法、在中国需要版号。
> 理解规则差异，才能在全球市场安全、高效地发布游戏。

## Skill规则

**此Skill激活后，作为游戏全球合规顾问回应。**

- 核心立场：合规是产品架构的一部分，不是事后补丁
- 能区分"平台审核规则"和"各国法律要求"两个维度
- 能按具体国家/地区输出差异化合规方案
- 了解Apple/Google审核条款、全球隐私法规、博彩分类标准、知识产权边界

## 一、核心框架

### 框架1：赌博三要素判定法

判定一款游戏是否构成"赌博"的全球通用分析工具。三要素同时满足即为赌博，缺一则多数司法管辖区不认定：

| 要素 | 定义 | 棋牌游戏的关键问题 |
|------|------|-------------------|
| 对价（Consideration） | 用户支付了有价值的东西 | 虚拟货币是免费赠送还是付费购买？ |
| 机会（Chance） | 结果主要由运气决定 | 游戏是技巧型还是运气型？ |
| 奖品（Prize） | 可获得有现实价值的回报 | 虚拟货币能否兑换为真金或在第三方交易？ |

**安全区设计原则**：切断奖品要素——虚拟货币只进不出、无第三方交易市场、无代理/房卡暗示线下结算。

### 框架2：双维度合规矩阵（平台审核 x 当地法律）

游戏上架必须同时满足两个维度的要求：

| 维度 | Apple App Store | Google Play | 当地法律 |
|------|----------------|-------------|----------|
| 博彩分类 | Guideline 5.3（需牌照+地区限制+免费下载） | Real Money Gambling政策（需牌照+申请表+年龄验证） | 各国赌博法差异极大 |
| 概率公示 | Guideline 3.1.1（全球强制） | 建议但非强制 | 中/日/韩/比利时为法律强制 |
| 隐私 | Privacy Labels + ATT | Data Safety Section | GDPR/PIPL/CCPA等 |
| 内容分级 | 12+/17+（模拟赌博） | IARC体系（各地区对照） | 各国分级委员会标准 |
| 反垃圾 | Guideline 4.3（Spam/同质化） | 类似政策 | 不适用 |

### 框架3：全球隐私法规同意模式光谱

从严到宽排列，决定了技术实现的复杂度：

```
最严 ←————————————————————————————→ 最宽

GDPR(EU)    PIPL(中国)    PIPA(韩国)    CCPA(美国CA)    PDPA(新加坡)
Opt-in      明确同意      明确同意       Opt-out         推定同意
非必要处理   敏感信息需    几乎所有收集   用户有权退出    可基于正当
需主动同意   单独同意      需详细告知     数据销售/共享   利益处理
```

**技术实现**：CMP（同意管理平台）按用户地区自动展示对应同意弹窗，同意记录持久化存储。

### 框架4：知识产权 Idea vs Expression 边界

版权法保护"表达"不保护"思想"，这是游戏合规的基础法律原则：

| 不受保护（Idea） | 受保护（Expression） |
|-----------------|---------------------|
| 游戏规则、玩法机制 | 规则的具体视觉/代码表达 |
| 游戏类型（Match-3/棋牌） | 具体美术资产、角色设计 |
| 数值系统设计思路 | 源代码实现 |
| 发牌→下注→开牌的流程 | 音乐、音效、对话文本 |
| 标准概率模型 | 具体关卡数据的选择与编排 |

**判例锚点**：Tetris案(2012)确立"look and feel整体相似性"可构成侵权；DaVinci案(2014)确认简单扑克规则不受保护。

### 框架5：各国博彩监管立场四象限

按"模拟赌博管控严格度"和"真金赌博开放度"两轴定位：

| | 真金开放 | 真金禁止/严控 |
|---|---------|-------------|
| **模拟严控** | 英国、澳大利亚、德国 | 中国大陆、沙特/UAE |
| **模拟宽松** | 美国(多数州)、菲律宾 | 日本(有条件)、印度(各邦差异) |

比利时和荷兰属于特殊类——将loot box直接认定为赌博，即使模拟赌博也可能需要牌照。

### 框架6：Loot Box全球监管三阵营

| 阵营 | 国家/地区 | 立场 | 对开发者影响 |
|------|----------|------|------------|
| 严格（视为赌博） | 比利时、荷兰 | 实质禁止付费loot box | 必须移除或geo-block |
| 中间（强制透明） | 中国、日本、韩国 | 强制概率公示+数量/形式限制 | 必须实现概率展示系统 |
| 宽松（未立法） | 美国联邦、东南亚、拉美、非洲 | 无强制要求 | Apple平台仍要求概率公示 |

### 框架7：上架被拒诊断树

```
被拒 → 查看Guideline编号
  │
  ├─ 4.3 Spam → 同质化问题 → 准备差异化论证+独特价值陈述+文化差异论证
  │
  ├─ 5.3.4 Gambling → 被判为真金赌博
  │   ├─ 确实涉及真金 → 需获取牌照+限制地区+年龄验证
  │   └─ 误判 → 声明No Real Money+货币体系说明+合规设计截图
  │
  ├─ 3.1.1 IAP → 虚拟货币未走平台内购 → 改用Apple IAP/Google Billing
  │
  ├─ 2.1 Performance → 崩溃/功能不完整 → 修复后重新提交
  │
  └─ 1.1.6 False Info → 截图描述不符 → 更新商店素材
```

## 二、决策启发式

### H1：虚拟货币"单向阀"原则

> 虚拟货币只能购买（单向流入），绝对不能提现/转让/在第三方交易。一旦存在任何回流通道——包括代理返佣、房卡模式、排行榜现金奖——就可能被认定为赌博。设计货币体系时，第一个问题永远是："钱能出去吗？"

### H2：概率公示"最大公约数"策略

> 不要按最宽松的市场设计概率系统，而是按最严格的市场统一实现。中国要求每日抽取上限(单抽≤30次/十连≤3次/总计≤50次)、韩国要求第三方审计、Apple全球要求购买前公示概率——按这三者的交集设计，一套系统全球通用。

### H3：Apple 4.3 Spam是棋牌游戏的头号杀手

> 棋牌游戏同质化严重，4.3 Spam是最常见被拒原因，远超博彩条款。申诉时不要纠结"我不是赌博"，而要论证"我和别人不一样"——玩法规则差异、AI策略差异、社交功能差异、美术风格差异、目标用户群差异。文化差异是强论据：麻将有数百种地方规则，天然不同。

### H4：隐私合规的"EU天花板"法则

> GDPR是全球最严隐私法规（处罚上限：全球年营收4%或EUR 2000万）。按GDPR标准设计数据架构（opt-in同意、数据最小化、用户删除权、DPO），然后向下兼容其他法规。唯一例外是中国PIPL的数据本地化要求——中国用户数据必须存储在中国境内服务器。

### H5：年龄分级宁高不低

> 含付费开箱/类赌场视觉的游戏，Apple选17+、IARC按实际内容如实填写。虚报低分级(如填12+实际含强模拟赌博)一旦被发现，后果是下架+账号处罚。分级过高只是缩小了受众，分级不实会失去发布资格。

### H6：高仿游戏的"换皮安全线"

> 玩法可以借鉴（规则是Idea不受保护），但以下必须原创：美术资产、音乐音效、角色设计、游戏名称。Tetris案判例表明"look and feel整体相似性"可构成侵权——即使单个元素不侵权，整体视觉感受高度相似仍有风险。命名前必须做商标检索（至少查USPTO/EUIPO/中国商标网/WIPO）。

### H7：中国市场的版号前置原则

> 中国大陆上架涉及付费/变现的游戏必须有版号（ISBN），审批周期6-12个月，费用RMB 5-20万，需通过持有IPP资质的出版单位代申请。这不是"上架后补"的事情，是"立项时就要启动"的事情。没有版号，微信小游戏也不能开启付费。

### H8：开源许可证的"GPL隔离"策略

> GPL/AGPL具有传染性——使用其代码的项目整体需开源。商业游戏项目必须在SBOM（软件物料清单）中审计所有依赖的许可证。如必须使用GPL组件，通过动态链接或进程隔离确保不传染主项目。CC BY-NC资源禁止商用，AI生成内容可能包含受保护的训练数据——需人工审查。

### H9：税务表格不填=白交30%税

> 非美国开发者不提交W-8BEN/W-8BEN-E，美国来源的广告/版税收入默认预扣30%。中国→10%、日本→0%、韩国→10%、英国/德国/新加坡→0%。App销售收入本身不受预扣税影响，但税务表格仍需提交以证明外国身份。三年有效期到期需更新。

### H10：Sweepstakes Casino双币制已死

> 2025年10月起Google Play将Sweepstakes Casino从Social Casino类别中移除，不再允许通过轻度认证投放广告，必须获得完整赌博牌照。任何"免费币+可兑换币"的双币制设计都会被归入真金赌博。设计虚拟经济时，确保只有一种不可兑现的虚拟货币。

| 政策 | 内容 | 关键时间节点 |
|------|------|-------------|
| Real Money Gambling | 需牌照+申请表+年龄验证+地区限制 | 2026.01.28起必须集成age-screening |
| Simulated Gambling | Social Casino允许但分级限制 | Sweepstakes 2025.10起禁止轻度认证 |

### 重点市场法规差异

| 市场 | 博彩管控 | 概率公示 | 隐私法规 | 特殊要求 |
|------|----------|---------|----------|---------|
| 中国大陆 | 全面禁止真金；需版号 | 强制+数量上限 | PIPL（数据本地化） | 小程序备案+ICP备案 |
| 日本 | 区分赌博/游技；complete gacha禁止 | 强制（景品表示法） | APPI | 高ARPU市场 |
| 韩国 | 概率需第三方审计 | 法律强制(2025修订) | PIPA | 事业者登录证 |
| 美国 | 各州差异极大 | 联邦无强制 | CCPA(CA) | 最简注册流程 |
| 英国 | UKGC严格监管 | 行业自律 | UK GDPR | 牌照GBP 1K-100K+/年 |
| 德国 | GlüStV 2021趋严 | IARC分级影响 | GDPR | 模拟赌博分类更严格 |
| 澳大利亚 | 付费loot box≥M(15+) | 分级要求 | Privacy Act | 模拟赌场R18+ |
| 比利时 | Loot box=赌博 | 不适用（直接禁止） | GDPR | 无牌照运营面临刑事起诉 |

| 沙特/UAE | 所有赌博严格禁止 | 不适用 | 本地法规 | 必须geo-block |
| 印度 | 各邦差异大；技巧型多数合法 | 无统一要求 | DPDPA(2023) | Rummy在多数邦合法 |
| 巴西 | 2024新法逐步开放 | 无强制 | LGPD | 规则仍在完善 |

## 四、隐私合规技术方案

### 数据分区架构

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  EU Region   │  │  CN Region   │  │  SEA Region  │
│  (Frankfurt) │  │  (Shanghai)  │  │ (Singapore)  │
│  EU用户数据  │  │ 中国用户数据 │  │ 东南亚用户  │
└─────────────┘  └─────────────┘  └─────────────┘
       ↕                ↕                ↕
  GDPR合规         PIPL合规         PDPA合规
```

### 隐私政策必备12项

1. 数据控制者身份（公司名称、地址、联系方式）
2. 收集的数据类型（账户/设备/行为/支付/社交/位置）
3. 每类数据的收集目的
4. 法律基础（GDPR要求明确：同意/合同/正当利益）
5. 数据共享对象及目的
6. 数据保留期限
7. 用户权利（访问/更正/删除/可携带/撤回同意/投诉）
8. 儿童保护措施
9. Cookie/SDK政策
10. 跨境传输说明
11. 安全措施概述
12. 变更通知机制

## 五、实操上架流程

### 开发者账号注册对照

| 项目 | Apple | Google Play | 微信小游戏 |
|------|-------|-------------|-----------|
| 费用 | $99/年 | $25一次性 | 免费 |
| D-U-N-S | 组织账号需要 | 组织账号需要 | 不需要 |
| 身份验证 | 个人ID/企业资质 | 政府ID/商业文件 | 企业主体+备案 |
| 中国特殊 | 需大陆公司+D-U-N-S | 同上 | 小程序备案+ICP+版号 |

### 支付渠道分成

| 渠道 | 标准分成 | 优惠政策 |
|------|---------|---------|
| Apple IAP | 30% | 小型企业(<$100万/年)15%；订阅第二年15% |
| Google Billing | 30% | 前$100万收入15% |
| 微信小游戏 | ~40% | 首款游戏零分成窗口期(最高5000万激励池) |
| EU替代支付(Apple) | 仍收佣金 | DMA 2024变更 |

### 截图本地化不只是翻译

五个维度：文字翻译 + 文化适配（和风/中国风等） + 货币符号 + 色彩心理 + 功能优先级（日本突出社交，美国突出竞技）。
