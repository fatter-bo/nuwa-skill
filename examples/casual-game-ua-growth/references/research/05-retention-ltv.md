# 研究维度五：留存与LTV

## 1. 留存基准（2025-2026）

### 行业整体基准

2025-2026年移动游戏留存率继续下滑，头部与尾部产品差距持续扩大。

**全品类中位数**：
- D1留存：约22%
- D7留存：约3.4-3.9%
- D30留存：数据持续恶化

**TOP25%项目**：
- D1留存：iOS 31-33%，Android 25-27%
- D7留存：7-8%

**TOP10%项目**：
- D1留存：约40%
- D7留存：约12%

**来源**：GameAnalytics - "2025 Mobile Gaming Benchmarks"；GameAnalytics - "Mobile retention benchmarks 2026"
**可信度**：高 — 行业权威基准报告，覆盖数万款游戏

### 按品类留存基准

| 品类 | D1（目标） | D7（目标） | D30（目标） |
|------|-----------|-----------|------------|
| 超休闲 | 20-30% | 3-5% | <2% |
| 休闲（Puzzle/Merge） | 35-45% | 12-18% | 6-10% |
| 中核休闲（Hybrid Casual） | 30-40% | 10-15% | 5-8% |
| 模拟/策略休闲 | 25-35% | 8-12% | 4-7% |

### 休闲游戏的关键留存节点

- **D1（次日留存）**：反映首次体验（FTUE）质量。40%+为优秀
- **D7（七日留存）**：反映核心循环深度和早期进度节奏
- **D30（月留存）**：LTV最准确的预测指标，从3%提升到10%可显著降低每留存用户成本
- **D90（季度留存）**：头部Merge游戏D90/D1相对留存率约25%

**来源**：Solsten - "The True Drivers of D1, D7, and D30 Retention in Gaming"；MAF - "Mobile Game Retention Benchmarks"
**可信度**：高 — 结合玩家心理研究和行业数据

## 2. LTV预测模型

### LTV核心公式

```
LTV = Σ(Dn留存率 × ARPDAUn)  [n从0到∞]
```

简化版：
```
LTV = ARPU(D_lifetime) = 累计每用户收入
```

### 常用LTV预测方法

#### 方法一：系数法（简单有效）

1. 从历史数据计算系数：K = D90 LTV / D3 LTV
2. 用新群组的实际D3 LTV × K = 预测D90 LTV
- **优点**：简单、不需要数据科学家
- **缺点**：假设用户行为模式稳定
- **适用**：标准品类的快速预估

#### 方法二：留存曲线拟合法

1. 收集D1-D7（或D14）的实际留存数据
2. 用幂函数/对数函数拟合留存曲线：Retention(d) = a × d^(-b)
3. 将预测留存曲线 × ARPDAU 求和得到LTV

#### 方法三：机器学习法

1. 使用用户早期行为特征（D0-D3）训练模型
2. 预测单用户LTV
3. 特征包括：首日会话数、首日付费、关卡进度、社交行为等
- **精度**：可达95%（足够数据条件下）
- **适用**：大规模运营的成熟产品

### ARPDAU基准

| 变现类型 | 健康ARPDAU |
|---------|-----------|
| 广告变现型（休闲） | >$0.15 |
| IAP主导型 | >$0.30 |
| 混合变现型 | >$0.20 |

### 东西方LTV方法论差异

- **西方标准**：Cohort LTV（群组累计ARPU），追踪特定群组的终身价值
- **东方标准**：ARPDAU（日活均收入），关注每日变现效率
- 两者互补：LTV看长期回报，ARPDAU看日常健康度

**来源**：Medium/Ruslan Valeev - "How to make retention model and calculate LTV"；Plarium - "Calculating LTV prediction for games"；AppAgent - "Complete Guide on Predictive LTV modeling"；Gamedeveloper.com - "How to calculate the lifetime value"
**可信度**：高 — 行业从业者和数据平台的方法论总结

## 3. 留存优化杠杆

### D1留存优化（首次体验）

| 杠杆 | 优化方向 | 预期影响 |
|------|---------|---------|
| FTUE（首次体验） | 30秒内让玩家完成核心操作并获得正反馈 | +5-15% D1 |
| 教学节奏 | 渐进式教学，边玩边学，避免长文本 | +3-8% D1 |
| 首日目标 | 提供明确的短期目标（完成X个关卡/解锁Y功能） | +2-5% D1 |
| 性能优化 | 减少加载时间、崩溃率 | +3-10% D1 |
| 推送许可引导 | 首日巧妙获取推送权限 | 间接影响D7+ |

### D7留存优化（核心循环深度）

| 杠杆 | 优化方向 | 预期影响 |
|------|---------|---------|
| 内容深度 | 确保7天内不会消耗完核心内容 | +3-8% D7 |
| 社交绑定 | 好友互赠、排行榜、协作任务 | +2-5% D7 |
| 每日登录奖励 | 递增式签到奖励，D5首次给硬货币 | +2-4% D7 |
| 推送策略 | 能量恢复、事件开始、建造完成推送 | +1-3% D7 |
| 限时活动 | 3-7天周期的限时事件 | +2-5% D7 |

### D30留存优化（长期驱动力）

| 杠杆 | 优化方向 | 预期影响 |
|------|---------|---------|
| Meta层深度 | 叙事推进、建造进度、收集系统 | +2-5% D30 |
| Battle Pass | 30天周期的双轨奖励系统 | +3-7% D30 |
| 社交沉没成本 | 公会/好友关系深化 | +2-4% D30 |
| LiveOps节奏 | 每周新内容/活动 | +2-5% D30 |
| 回流机制 | 离线奖励、回归礼包 | 降低流失率 |

### 回流与再激活

**回流礼包梯度设计**：
- 1-3天离线：轻度（少量能量+软货币）
- 3-7天离线：中度（大量能量+加速道具）
- 7-14天离线：重度（硬货币+限时专属包）
- 14天+离线：重新引导（等同新用户流程）

**推送通知策略**：
- 能量恢复满时推送
- 限时活动开始时推送
- 建造/生产完成时推送
- 2-3天未登录时推送
- 好友请求帮助时推送

**可信度**：高 — 结合行业数据和产品实操经验

## 4. LTV与UA的关系

### 盈利公式

```
利润 = LTV - CPI - 运营成本
```

当 LTV > CPI 时，UA投放盈利。

### ROAS与LTV的关系

```
D_n ROAS = D_n LTV / CPI × 100%
```

### 关键决策框架

| 信号 | 含义 | 行动 |
|------|------|------|
| D7 ROAS > 目标 | 渠道/创意质量好 | 放量 |
| D7 ROAS < 目标 但 D30 ROAS 达标 | 后端变现强 | 保持并优化前端 |
| LTV/CPI > 1 但利润薄 | 需要优化两端 | 同时优化CPI和变现 |
| LTV/CPI < 1 | 亏损 | 暂停该渠道/创意 |
