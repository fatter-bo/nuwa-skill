# 研究维度四：交叉推广

## 1. 交叉推广基础

### 定义与价值

交叉推广（Cross-Promotion）是指在一款游戏内推广同一工作室/发行商旗下的其他游戏，实现用户在产品组合内的流转。

**核心价值**：
- **零成本获客**：内部流量无需支付CPI
- **高预测性**：比外部买量更可控、更稳定
- **用户质量高**：已验证的游戏用户，安装和留存质量优于外部流量
- **LTV最大化**：即使玩家流失于A游戏，也可在B游戏继续贡献收入
- **LiveOps协同**：可与运营活动配合，精准控制推广节奏

**前提条件**：工作室拥有2款以上活跃产品

**来源**：TyrAds - "Mobile Game Cross-Promotion"；MAF - "Mobile Game Cross-Promotion: More Installs, Zero Costs"
**可信度**：高 — 行业营销平台基于实操经验

## 2. 广告位设计

### 广告位类型

| 位置 | 形式 | 打扰度 | 转化率 | 适用场景 |
|------|------|--------|--------|---------|
| 主界面Banner | 底部/侧边常驻横幅 | 低 | 低 | 持续曝光 |
| 关卡间Interstitial | 全屏插屏 | 高 | 中-高 | 自然断点 |
| 成就达成弹窗 | 半屏推荐卡片 | 中 | 高 | 情绪高点 |
| 新闻/公告区 | 嵌入式卡片 | 低 | 低-中 | 信息流 |
| 加载页 | 全屏展示 | 低 | 中 | 等待时间利用 |
| 原生UI嵌入 | 游戏内原生按钮/图标 | 最低 | 中 | 无缝体验 |

### 最佳触发时机

**核心原则**：在玩家即将离开游戏之前展示交叉推广——此时推广另一款游戏而非让玩家流失到竞品。

**高转化时机**：
1. **关卡完成/里程碑达成后**：正向情绪+自然断点
2. **能量耗尽时**：等待恢复期间的替代选择
3. **连续失败后**：提供"换个口味"的选择
4. **会话结束时**：玩家主动退出前的最后触点
5. **每日奖励领取后**：完成日常任务的满足感节点

**来源**：AppAgent - "Cross-Promotion Guide for Mobile Game"；AppSamurai - "Mobile Game Cross Promotion Tips 2025"
**可信度**：高 — UA服务商基于大量案例分析

## 3. 交叉推广策略

### 策略一：品类相近推广

推广品类/美术风格/目标用户相似的游戏。

- **优势**：用户偏好匹配度高、转化率最佳
- **风险**：可能蚕食现有游戏用户（自我竞争）
- **适用**：用户已进入流失期的游戏→推荐新游戏

### 策略二：品类互补推广

推广不同品类但共享用户画像的游戏。

- **优势**：满足用户不同需求，减少自我竞争
- **案例**：合成游戏→三消游戏、解谜游戏→字谜游戏
- **关键**：确保美术风格和品质感一致

### 策略三：新品冷启动

利用成熟游戏的流量为新游戏导流。

- **流量分配**：从日活最高的游戏导出5-10%流量到新品
- **激励机制**：在新品达成目标后回到旧游戏领取奖励
- **注意**：不能过度抽血影响成熟产品的留存和收入

### 策略四：双向导流

两款游戏互相推广，共同受益。

- **联动活动**：跨游戏事件，在A和B完成各自任务获得联合奖励
- **共享IP/角色**：同一角色在不同游戏中出现
- **数据打通**：共享用户画像，精准推荐

**来源**：The Game Marketer - "7 Essential Cross-Promotion Strategies"；RocketBrush - "Mobile Game Marketing 2025"
**可信度**：高 — 行业营销专家基于成功案例

## 4. 用户推荐算法

### 基础推荐逻辑

```
推荐分数 = 品类偏好匹配度 × 用户活跃阶段权重 × 目标游戏质量分 × 时效性加权
```

### 用户分层推荐

| 用户阶段 | 特征 | 推荐策略 |
|---------|------|---------|
| 新用户（D1-D7） | 刚建立游戏习惯 | 不推荐（保护留存） |
| 活跃用户（D7-D30） | 稳定参与、熟悉品牌 | 轻度曝光、品牌认知 |
| 成熟用户（D30+） | LTV确定、品牌忠诚 | 主要推广对象 |
| 流失预警用户 | 活跃度下降 | 积极推广替代产品 |
| 流失用户 | 已停止活跃 | 通过推送/邮件推荐新品 |

### 个性化推荐信号

1. **游戏偏好**：历史游玩的品类/题材/玩法
2. **消费行为**：付费用户推荐高变现产品、免费用户推荐广告变现产品
3. **社交图谱**：好友在玩的游戏
4. **设备特征**：高端设备用户可推荐重度游戏
5. **地域偏好**：不同市场推荐本地化程度高的产品

### 频控与体验保护

- **单日展示上限**：2-3次交叉推广展示
- **冷却期**：同一游戏推广间隔≥24小时
- **负反馈关闭**：用户明确关闭后7天内不再展示同一推广
- **A/B测试**：持续测试推广频率对源游戏留存的影响

**来源**：Stepico - "Mobile Game Marketing Strategy in 2026"；MobileAction - "7 winning game app marketing strategies"
**可信度**：中-高 — 基于行业最佳实践，具体算法因公司而异

## 5. 效果度量

### 核心指标

| 指标 | 定义 | 基准 |
|------|------|------|
| 展示→点击率 | 看到推广→点击 | 2-5% |
| 点击→安装率 | 点击→安装目标游戏 | 15-30% |
| 交叉推广CPI | 虚拟CPI（机会成本） | $0（直接成本） |
| 目标游戏D7留存 | 交叉推广用户的7日留存 | 通常高于付费买量用户 |
| 源游戏留存影响 | 推广是否影响源游戏留存 | 影响应<1% |
| Portfolio LTV | 用户在整个产品组合的总LTV | 核心追踪指标 |

### Portfolio LTV思维

传统UA以单款游戏计算ROAS，交叉推广要求以**产品组合**为单位计算：

```
Portfolio LTV = 游戏A的LTV + 交叉推广转化概率 × 游戏B的LTV + ...
```

即使单款游戏LTV不足以覆盖CPI，通过交叉推广延长用户在组合内的生命周期，Portfolio LTV可能盈利。

**来源**：Segwise - "Essential Mobile Game Marketing Strategies"；NetMarvel - "Mobile Game Marketing Strategies 2025"
**可信度**：中-高 — 行业方法论框架
