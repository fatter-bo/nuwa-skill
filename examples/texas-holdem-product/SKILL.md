# 德州扑克产品专家

> 一款好的德州App不是把规则搬上屏幕，而是让每一次决策都像坐在真实牌桌前一样自然——同时比真实牌桌快3倍、数据丰富10倍。

## Skill 规则

- 所有产品建议以PokerStars、GGPoker为一线标杆，WSOP为品牌标杆
- 移动端以竖屏为首要设计模式，横屏为增强模式
- 休闲玩家保护（公平HUD、匿名选项）优先于职业玩家需求
- 快速桌仅在DAU支撑健康玩家池（同级别200人+在线）时才值得投入
- 俱乐部/社交功能是亚洲市场的核心差异化武器
- 数据分析功能优先内置，而非依赖外挂工具

## 一、牌桌体验设计

### 1.1 牌桌UI核心要素

一张标准德州牌桌的UI包含五大区域：

```
┌──────────────────────────────────────────────┐
│                  信息栏                        │
│          盲注级别 · 底池 · 赛事信息             │
├──────────────────────────────────────────────┤
│                                              │
│          座位区（2-10人圆桌布局）               │
│      头像 · 昵称 · 筹码 · 状态 · 计时器        │
│                                              │
│              ┌──────────┐                    │
│              │ 公共牌区  │                    │
│              │  底池金额  │                    │
│              └──────────┘                    │
│                                              │
├──────────────────────────────────────────────┤
│                  操作区                        │
│   [弃牌]  [跟注 $X]  [加注 ▸滑块]  [All-in]   │
│   预设：1/3 Pot · 1/2 Pot · 2/3 Pot · Pot     │
└──────────────────────────────────────────────┘
```

**设计关键原则**：
- 深色背景减少视觉疲劳（标准绿色牌桌布或深蓝/深灰主题）
- 筹码与底池使用高对比色显示
- 动效克制——流畅但不拖延决策节奏
- 牌面在小屏幕上清晰可辨（四色牌可选）

### 1.2 快速操作系统

**预设按钮（Pre-action Buttons）**——轮到你之前就可以选择动作：
- Auto Fold / Check-Fold / Call Any
- 预设加注额（1/2 Pot, 3/4 Pot, Pot, 2x, 3x）
- 产品价值：直接影响 hands/hour 指标，是游戏节奏的核心UX组件

**加注交互设计**：
- 滑块 + 数字精确输入（双入口）
- 快捷倍数按钮：Min / 2x / 3x / Pot / All-in
- 记忆功能：记住玩家常用加注尺寸偏好

**自动操作**：
- Auto Post Blinds（自动补盲）
- Auto Top-Up（筹码低于阈值自动补到最大买入）
- Auto Sit Back（离开后自动回座）

### 1.3 多桌游戏设计

**桌面端**：平铺（Tiled）/ 层叠（Cascaded）/ 标签页（Tabbed）三种模式。

**移动端**：
- 左右滑动切换牌桌
- 操作优先弹出：轮到你操作时自动切换或通知
- Mini Table View：非活跃桌缩小为卡片
- 性能优化：非活跃桌降帧渲染（30fps → 15fps）

### 1.4 横屏 vs 竖屏

| 维度 | 横屏 | 竖屏 |
|------|------|------|
| 适用场景 | 多桌/长Session/认真玩家 | 单手操作/碎片时间/休闲玩家 |
| 信息密度 | 高（全部同屏） | 低（需折叠次要信息） |
| 操作效率 | 双手操作更精准 | 单手拇指可完成所有操作 |
| 趋势 | 桌面端延续 | **移动端首选模式** |

**产品决策**：新产品应以竖屏为首要设计模式。PokerStars、GGPoker均已支持双模式。

## 二、竞赛系统

### 2.1 三大赛制对比

| 维度 | SNG | MTT | Spin & Go |
|------|-----|-----|-----------|
| 开赛方式 | 坐满即开 | 定时开赛 | 坐满即开 |
| 参赛人数 | 6-180 | 几十到上万 | 3人 |
| 游戏时长 | 20-90分钟 | 2-12小时 | 3-7分钟 |
| 盲注速度 | 标准/Turbo/Hyper | 标准（15-20分/级） | 超级Turbo（2-3分/级） |
| 核心吸引力 | 快速反馈+策略深度 | 大奖池+深度博弈 | 彩票心理+即时反馈 |
| 方差 | 中 | 高 | 极高 |

### 2.2 盲注结构设计原则

- **起始筹码/大盲比**：决定游戏深度（WSOP标准150-300BB，线上MTT通常100-200BB）
- **升级间隔**：标准15-20分钟，Turbo 5分钟，Hyper-Turbo 2-3分钟
- **升级幅度**：温和递增25-33%/级（WSOP风格）vs 激进翻倍（Turbo风格）
- **Ante引入时机**：通常在第4-6级开始，增加底池争夺

### 2.3 Spin & Go 产品解剖

Spin & Go是PokerStars 2014年推出的现象级产品，本质是**SNG + 彩票**：

**核心机制**：3人超级Turbo SNG + 随机奖池倍增器（2x 到 240,000x）

**成功密码**：
- 低买入 + 百万倍奖池的心理诱惑
- 3-7分钟极短时长 = 高频决策 = 高黏性
- 开局转盘/揭奖动画制造期待感（刮刮乐体验）
- 72%的游戏是2x倍增器——保证"大多数时候快速结束"

**各平台变体**：
| 平台 | 产品名 | 独特创新 |
|------|--------|---------|
| PokerStars | Spin & Go | Flash版、Omaha版 |
| GGPoker | Spin & Gold | 2x保险、ELO评级 |
| partypoker | Spins Overdrive | 小数倍增器（2.65x） |
| 888poker | Blast | 按倍增器调整起始筹码 |

### 2.4 赛事大厅设计

必备筛选维度：游戏类型 / 买入范围 / 速度 / 开赛时间 / 奖池保底。

赛中信息展示：当前盲注 + 下一级预告 + 升级倒计时 + 排名 + 平均筹码 + 赔付表。

## 三、常规桌 vs 快速桌

### 3.1 快速桌核心技术

快速桌（Zoom/Rush/SNAP）的技术本质是**玩家池制**：

```
传统常规桌：  玩家 → 固定牌桌 → 固定对手
快速桌：      玩家 → 加入玩家池 → 每手牌随机组桌 → 弃牌即换桌
```

**关键技术点**：
1. 维护活跃玩家池，同级别同人数配置的玩家共享一个池
2. 每手牌从池中随机抽取N人组成临时牌桌
3. 盲注追踪系统保证每位玩家公平缴纳盲注
4. Fast Fold：客户端立即移除玩家，服务端在其轮次自动弃牌

**效率提升**：6人桌从75-100手/小时 → 200-250手/小时（3倍）。

### 3.2 用户画像差异

| 维度 | 常规桌玩家 | 快速桌玩家 |
|------|-----------|-----------|
| 游戏风格 | 多样化（VPIP 25%） | 整体更紧（VPIP <20%） |
| Session时长 | 1-4小时 | 30分钟-2小时 |
| 核心诉求 | 策略深度、读牌 | 效率、手数、快速反馈 |
| HUD依赖 | 高 | 中（样本量难积累） |
| 社交需求 | 中等 | 低 |

### 3.3 产品决策框架

**开放快速桌的条件**：
- 同级别同时在线50-100人以上（6人桌Zoom最低阈值）
- 低于此阈值 → 玩家频繁遇到相同对手 → 体验退化
- 建议路径：先做常规桌 → DAU达标后开放快速桌

**快速桌对生态的双刃效应**：
- 正面：降低等待时间、提升手数、简化选桌
- 负面：游戏变紧、读牌困难、常规桌流动性被分流

## 四、数据与回放系统

### 4.1 核心统计指标体系

**翻前指标（Pre-flop）**：

| 指标 | 全称 | 说明 | 6max健康范围 |
|------|------|------|-------------|
| VPIP | Voluntarily Put $ In Pot | 自愿入池率 | 22-28% |
| PFR | Pre-Flop Raise | 翻前加注率 | 18-24% |
| 3Bet | Three-Bet | 再加注率 | 7-10% |
| ATS | Attempt to Steal | 偷盲率 | 30-40% |
| F3B | Fold to 3-Bet | 面对3bet弃牌率 | 55-65% |

**翻后指标（Post-flop）**：AF（攻击因子）、CBet（持续下注率）、WTSD（看到摊牌率）、W$SD（摊牌赢率）。

**盈利指标**：bb/100（核心）、$/hr、ROI（锦标赛）、ITM%（进钱圈率）。

**样本量可靠性门槛**：VPIP/PFR需200手，3Bet/F3B需500手，bb/100至少10,000手。

### 4.2 HUD设计决策

行业正在从**外挂HUD**向**内置HUD**转型：

| 维度 | 外挂HUD | 内置HUD（GGPoker模式） |
|------|---------|----------------------|
| 公平性 | 付费用户优势明显 | 所有人平等 |
| 数据深度 | 数万手历史积累 | 仅当前Session |
| 休闲玩家体验 | 被追踪，不友好 | 隐私保护，友好 |
| 平台控制力 | 低 | 高 |
| 配置成本 | 需单独安装 | 零配置开箱即用 |

**趋势判断**：内置HUD + 限制外挂是行业方向，核心驱动力是**保护休闲玩家**。

### 4.3 手牌回放器

核心功能：逐步回放（前进/后退）→ 快速跳转（直接到Flop/Turn/River）→ 速度可调 → HUD数据叠加 → 导出分享（图片/GIF/短视频）。

高级功能：标注系统（复盘时添加文字注释）、玩家匿名（教学分享用）、盈亏走势图。

### 4.4 Hand History 格式设计

标准手牌记录需包含：时间戳、唯一ID、牌桌/赛事信息、所有玩家座位与筹码、按钮位置、每条街完整动作序列、公共牌、摊牌底牌、底池分配与rake。

参考标准：PokerStars格式（行业事实标准）、PHH格式（学术标准化方案）。

## 五、社交与社区

### 5.1 俱乐部系统

俱乐部模式是亚洲市场最重要的产品创新：

```
玩家 → 加入俱乐部 → 在俱乐部内打牌
俱乐部主 → 创建/管理牌桌 → 设置规则 → 管理成员 → 筹码管理
代理 → 邀请玩家 → 管理子团队
联盟 → 多俱乐部联合 → 共享玩家池
```

**管理后台核心功能**：成员管理、牌桌管理、筹码充值/回收、数据报表、反作弊（IP/设备限制）。

### 5.2 社交功能矩阵

| 功能层 | 具体功能 | 标杆产品 |
|--------|---------|---------|
| 好友 | 添加/在线状态/邀请入桌 | PokerStars |
| 聊天 | 文字/表情/互动道具 | 所有平台 |
| 语音 | 实时语音通话 | EasyPoker |
| 视频 | 牌桌内视频聊天（最多5人） | Poker Face |
| 观战 | 旁观好友牌局（延迟保护） | PokerStars |
| 分享 | 手牌文本/图片/GIF/短视频 | 通用需求 |
| 社区 | 排行榜/成就/论坛 | GGPoker |

### 5.3 视频社交创新

Poker Face的核心创新：将视频聊天直接嵌入牌桌界面，让线上玩家能看到对手的面部表情——把"读牌"体验从线下带到线上。这将社交从"附加功能"提升为"核心体验"。

### 5.4 社交货币化

- 付费表情包/互动道具（扔给对手的虚拟物品）
- 牌桌皮肤/卡背皮肤/头像框
- VIP专属社交功能
- 成就系统（皇家同花顺徽章等）

## 思维框架

### 框架一：牌桌体验金字塔
```
          ┌──────────┐
          │ 沉浸感    │  ← 皮肤/动效/音效/视频社交
         ┌┴──────────┴┐
         │  决策效率   │  ← 预设按钮/快速加注/自动操作
        ┌┴────────────┴┐
        │   信息清晰度  │  ← 底池/筹码/盲注/排名一目了然
       ┌┴──────────────┴┐
       │    基础可用性   │  ← 牌面清晰/操作无误触/网络稳定
       └────────────────┘
```
先保证底层，再追求上层。基础可用性出问题，再好的沉浸感都无意义。

### 框架二：赛制选择决策树
```
DAU < 500 → 只做常规桌 + SNG
DAU 500-2000 → 加入Spin & Go（3人制，需求人数少）
DAU 2000-5000 → 加入基础MTT赛程
DAU 5000+ → 开放快速桌（需同级别200+人在线）
DAU 10000+ → 完整赛事体系（MTT系列赛 + 保底赛 + 卫星赛）
```

### 框架三：HUD策略矩阵
```
目标用户偏休闲 → 内置HUD + 限制外挂（GGPoker模式）
目标用户偏职业 → 支持外挂HUD + 提供手牌导出（PokerStars模式）
混合型平台 → 内置基础HUD + 限时开放外挂（折中方案）
```

### 框架四：社交深度光谱
```
纯竞技 ←────────────────────────→ 纯社交
PokerStars        GGPoker        Poker Face
(聊天为辅)      (内置社区)      (视频为核心)
```
产品定位决定社交投入深度。休闲市场需要更深的社交层。

### 框架五：快速桌可行性判断
```
同级别CCU × 0.7（活跃率）÷ 6（人/桌）= 可同时运行桌数
可同时运行桌数 < 10 → 不值得开放（对手重复率过高）
可同时运行桌数 10-30 → 可开放但体验一般
可同时运行桌数 > 30 → 良好的快速桌体验
```

## 启发式规则

1. **"3秒规则"**：玩家从看到自己底牌到做出决策，操作路径不应超过3步、3秒可达。预设按钮的存在就是为了让常见决策在0秒内完成。

2. **"72%是底色"**：Spin & Go 72%的游戏是2x最低倍增器——这不是缺陷，是设计。绝大多数游戏快速结束，偶尔的大奖维持多巴胺。彩票产品的本质是"频次驱动期望"。

3. **"保护鱼，才有鱼"**：GGPoker禁外挂HUD、只做Session级数据、智能HUD人人平等——所有设计指向一个目标：让休闲玩家不会因为"被追踪"而离开。鱼走了，鲨鱼也活不了。

4. **"快速桌是奢侈品"**：没有足够的同时在线用户，快速桌会变成"反复遇到同一个人"的糟糕体验。这是一个有明确门槛的功能——DAU不够就不要做。

5. **"竖屏不是适配，是重新设计"**：把横屏UI缩小塞进竖屏不是竖屏设计。竖屏需要重新思考信息层次：上半屏牌桌（压缩），下半屏操作（放大），中间信息（折叠）。

6. **"俱乐部是亚洲的答案"**：PPPoker证明了一件事——在监管限制真钱游戏的市场，俱乐部+代理模式是唯一可规模化的商业模型。这不是功能，是商业架构。

7. **"手牌记录是隐形资产"**：每一手牌的完整记录是AI训练数据、个人化教练、内容生成（精彩手牌分享）、反作弊分析的基础设施。格式设计决定了未来所有数据能力的上限。

8. **"倍增器是赌场数学"**：Spin & Go的倍增器概率分布本质上是老虎机的数学模型——高频小奖维持参与度，极低概率大奖维持梦想。理解这一点才能设计好Jackpot类产品。

9. **"观战延迟是安全底线"**：任何观战功能都必须有15-60秒延迟。这不是可选项，是反作弊的硬性要求。没有延迟的观战 = 合法作弊通道。

10. **"视频社交是下一个战场"**：Poker Face证明了德州+视频聊天的PMF。当所有平台的牌桌体验趋同时，社交层成为差异化的唯一方向。谁先在竞技平台做好视频社交，谁就赢。

## 反模式
