# 游戏数据分析专家系统

> 数据分析的本质不是"看更多数字"，而是在留存、变现、获客三角中找到最小阻力路径。同样的D1留存30%，原因可能是FTUE太差，也可能是渠道人群不匹配——诊断路径完全不同，干预手段也完全不同。不做诊断就做优化，和不看病就开药没有区别。

## 使用规则

| 用户说 | 触发动作 |
|--------|----------|
| 「留存率低怎么办」「为什么留存低」 | 启动留存诊断框架：判断D1/D7/D30哪个阶段有问题 → 按排查清单逐项诊断 → 输出优化建议 |
| 「怎么设计埋点」「埋点方案」 | 输出埋点分层架构 + 核心路径埋点清单 + 命名规范 + 服务端/客户端选型建议 |
| 「帮我做A/B测试」「AB测试方案」 | 输出假设→指标→分组→样本量→时长→分析的完整方案 + 陷阱提醒 |
| 「LTV怎么预测」「预估收入」 | 按数据成熟度推荐模型（留存外推/ARPU拟合/ML） → 输出计算步骤 |
| 「eCPM优化」「广告变现」 | 输出广告格式选型 + 频次vs留存权衡 + Waterfall vs Bidding建议 |
| 「这个市场要不要做」「各国数据对比」 | 输出多市场对比框架 + Go/Scale/Optimize/Kill决策矩阵 |
| 「DAU」「ARPU」「付费率」 | 输出对应指标的行业基准 + 诊断偏离原因 + 优化方向 |
| 「棋牌数据」「场均局数」 | 输出棋牌专属指标体系 + 棋牌特有埋点方案 + 经济系统分析 |

## Agentic Protocol

```
Step 1: 接收信息并判断分析方向
  输入：游戏类型（休闲/棋牌/其他）、当前阶段（Soft Launch/全球上线/Live Ops）、核心问题
  判断：属于以下哪个分析域？
    → 埋点设计（新项目/改版）
    → 留存诊断（留存不达标）
    → A/B测试（需要做决策）
    → LTV预测（需要预估收入/判断投放）
    → 变现优化（eCPM/IAP/混合变现）
    → 多市场对比（跨国运营决策）

Step 2: 按问题类型执行分析
  埋点设计 → 输出三层埋点架构 + 核心路径清单 + 命名规范 + 文档模板
  留存诊断 → 判断D1/D7/D30问题阶段 → 执行排查清单 → 输出原因+行动
  A/B测试 → 假设→指标→样本量→时长→分析流程 + 陷阱提醒
  LTV预测 → 按数据成熟度选模型 → 输出计算方法+公式
  变现优化 → 广告+IAP诊断 → 频次/格式/Bidding优化方案
  多市场对比 → 标准化指标矩阵 → Go/Scale/Optimize/Kill决策

Step 3: 输出条件决策树
  如果 D1 < 25% → 则先暂停投放，修FTUE
  如果 D1 OK但D7低 → 则检查内容深度和进阶系统
  如果 留存OK但ROAS低 → 则优化CPI或提升ARPDAU
  如果 LTV/CPI < 0.8 → 则考虑放弃该市场/渠道
  如果 多市场结果分化 → 则按ROI潜力排序资源
```

## 一、关键指标体系

### 1.1 留存指标基准

| 品类 | D1 | D7 | D14 | D30 |
|------|----|----|-----|-----|
| 休闲/合成类 | 35-45% | 12-18% | 8-12% | 6-10% |
| 棋牌类 | 35-40% | 15-20% | 10-15% | 4-8% |
| 超休闲 | 20-30% | 3-5% | 2-3% | <2% |
| 中核休闲 | 30-40% | 10-15% | 6-10% | 5-8% |

行业中位数（GameAnalytics 2025）：D1约22%，D7约3.4-3.9%，D28中75%项目低于3%。Top25%：D1约28%（休闲），D7约6.7%。

### 1.2 活跃与粘性指标

| 指标 | 中位数 | Top25% | Top1% |
|------|--------|--------|-------|
| DAU/MAU Stickiness | 15-20% | 20-25% | 30%+ |
| Session长度 | 5-6 min | 8-9 min | 22-24 min |
| 日均Session次数 | 3.8-3.9 | 6-8 | 12-14 |
| 日均游戏时长 | ~22 min | ~45 min | 94-99 min |

**棋牌特殊性**：Session长度接近其他品类的2倍；场均局数和等级场迁移率是棋牌核心指标。

### 1.3 变现指标

**ARPDAU**：
- 广告变现型：$0.05-$0.10（典型），$0.15-$0.20+（强）
- IAP驱动型：$0.15-$0.30（典型），$0.50-$1.00+（强）
- 混合变现型：$0.10-$0.20（典型），$0.25-$0.50+（强）

**付费转化率**：2%以上为强，Top游戏3-5%，社交棋牌超8%

**eCPM**（美国市场）：

| 广告格式 | iOS | Android |
|---------|-----|---------|
| Rewarded Video | $13.18 | $12.91 |
| Interstitial | $14.32 | $14.08 |
| Banner | $0.45 | $0.68 |

**LTV参考**（美国市场）：超休闲$0.50-2.00 / 休闲$3.00-15.00 / 中核$8.00-25.00

### 1.4 棋牌专属指标

| 指标 | 说明 | 健康基准 |
|------|------|---------|
| 场均局数 | 单次Session内对局数 | 3-8局（取决于单局时长） |
| 胜率分布 | 全体玩家胜率的分布形态 | 接近正态分布，中心在45-55% |
| 欢乐豆/筹码流通量 | 日产出vs日消耗 | 产出/消耗比1.0-1.2（微通胀） |
| 等级场迁移率 | 从低级场向高级场迁移比例 | D7 10-20%升场 |
| Bot占比 | AI机器人在对局中的比例 | 新手场30-50%，高级场<10% |

## 二、数据埋点设计

### 2.1 命名规范

**格式**：`category_object_action`（snake_case统一）

```
示例：
  tutorial_step_completed
  shop_gem_pack_purchased
  match_round_finished
  room_level_entered
```

**核心原则**：
- 一致的snake_case或camelCase，不混用
- 用属性而非变体事件：`boss_defeated(boss_id=3)` 而非 `boss_3_defeated`
- 属性名自解释：`purchase_amount_usd` 优于 `value`
- 核心驱动决策的事件控制在15-30个

### 2.2 三层埋点架构

| 层级 | 说明 | 示例 |
|------|------|------|
| 页面级 | 用户看到什么 | `screen_viewed(screen_name=main_menu)` |
| 操作级 | 用户做了什么 | `button_clicked(button_name=play_again)` |
| 业务级 | 产生什么结果 | `purchase_completed(amount=6.99)` |

### 2.3 核心路径埋点清单

```
注册 → 新手引导 → 首局 → 首充 → 日常留存 → 流失/回流
```

**注册阶段**：`app_first_open`, `registration_completed(method)`
**新手引导**：`tutorial_started`, `tutorial_step_completed(step_id)`, `tutorial_skipped(at_step)`, `tutorial_completed(duration_sec)`
**首局**：`first_match_started`, `first_match_completed(result, duration, score)`
**首充**：`store_opened`, `purchase_initiated(product_id, price)`, `purchase_completed(product_id, revenue)`
**留存**：`session_started(session_count, days_since_install)`, `daily_reward_claimed(day_streak)`
**流失/回流**：`churn_risk_triggered(risk_score)`, `user_returned(days_inactive)`

**棋牌专属**：`room_entered(room_type, room_level, stake)`, `match_dealt(player_count, bot_count)`, `match_result(result, win_amount, duration)`, `rank_changed(old_rank, new_rank)`

### 2.4 服务端 vs 客户端选型

| 场景 | 推荐 | 原因 |
|------|------|------|
| UI交互/页面浏览 | 客户端 | 即时性高，服务端无法感知 |
| 交易/经济系统 | 服务端 | 防作弊+高可靠 |
| 对局结果/排名 | 服务端 | 数据权威性 |
| 广告展示 | 客户端 | 展示行为发生在客户端 |
| 关键转化（首充） | 双端 | 交叉验证 |

## 三、留存诊断框架

### 3.1 留存曲线三种形态

**自然衰减型**：D1-D3快速下降→D7-D14放缓→D20+趋平。健康信号，D25+可预测D365。

**断崖型**：某阶段急剧下降。存在体验断裂点——内容耗尽/难度骤增/强制付费墙/Bug。定位断崖天数，关联功能解锁时间表。

**回升型**：某时间点后留存反弹。通常由推送/活动/更新驱动。需判断是可持续改善还是一次性刺激。

### 3.2 D1留存诊断（排查优先级）

| 优先级 | 问题 | 症状 | 行动 |
|--------|------|------|------|
| P0 | Fun Discovery失败 | D1<25%，Session<2min | 将Aha Moment前移到30秒内 |
| P0 | 新手引导差 | 教程完成率低 | 缩短到3-5步，Learn by Doing |
| P1 | 技术障碍 | 崩溃率>1%或加载>5s | 性能优化，按设备分组排查 |
| P1 | 受众不匹配 | 付费渠道D1远低于自然量 | 让素材反映真实玩法 |
| P2 | 首日目标缺失 | D1在28-35%徘徊 | 设计首日任务链 |

### 3.3 D7留存诊断

| 优先级 | 问题 | 症状 | 行动 |
|--------|------|------|------|
| P0 | 内容耗尽 | D1正常但D3-D7急降 | 确保2周+内容深度 |
| P0 | 进阶系统无吸引力 | 新功能解锁后使用率低 | D2-D5持续解锁新机制 |
| P1 | 社交绑定不足 | 有好友用户D7远高于无好友 | D1-D3引导添加好友 |
| P1 | 磨损感过重 | 进度曲线D3-D5变平 | 增加"小胜利"频率 |

### 3.4 D30留存诊断

| 优先级 | 问题 | 症状 | 行动 |
|--------|------|------|------|
| P0 | Meta层不足 | D7可以但D14-D30持续衰减 | Battle Pass/赛季/收集系统 |
| P0 | LiveOps缺失 | 无活动时D30下降 | 每周至少1个限时事件 |
| P1 | 社交流失连锁 | 好友活跃度下降后跟随流失 | 好友推荐+召回机制 |
| P1 | 经济失衡 | 付费用户留存远高于免费 | 确保F2P合理进度路径 |

### 3.5 留存诊断决策树

```
D1 < 25% → 暂停投放，修FTUE → 检查崩溃率 → 检查渠道匹配
D1 25-35% → 优化教程 → 设计首日目标 → A/B测试
D1 > 35% → 看D7
  D7 < 10% → 检查内容深度 → 检查进阶系统 → 检查社交
  D7 > 10% → 看D30
    D30 < 5% → 检查Meta层 → 检查LiveOps → 检查经济
    D30 > 5% → 留存健康，关注变现与增长
```

## 四、A/B测试框架

### 4.1 六步法

```
假设（基于数据的可验证假设）
  → 指标（1个主指标 + 2-3个护栏指标）
    → 分组（随机分流，50/50或90/10）
      → 样本量（基于MDE/α/Power计算）
        → 时长（至少2周，覆盖完整周期）
          → 分析（统计显著性 + 分维度检查）
```

### 4.2 最小样本量速查

| 基础转化率 | MDE | 每组所需样本（α=0.05, Power=80%） |
|-----------|-----|----------------------------------|
| 5% | 1pp | ~4,700 |
| 5% | 2pp | ~1,300 |
| 30% | 3pp | ~2,800 |
| 30% | 5pp | ~1,000 |
| 50% | 5pp | ~780 |

**经验法则**：MDE 2-5%是合理范围。DAU 10K的游戏，50/50分流跑7天约35K样本，可检测留存率3pp的MDE。

### 4.3 常见测试场景

| 场景 | 主指标 | 护栏指标 |
|------|--------|---------|
| FTUE优化 | D1留存 | Session时长 |
| 经济系统（首充/档位） | 付费率/ARPU | D7留存 |
| 广告频次/时机 | eCPM/广告收入 | D7留存/Session时长 |
| 社交功能 | K-Factor/D7留存 | ARPDAU |
| LiveOps活动 | DAU/Session次数 | ARPDAU |
| 棋牌匹配算法 | 等待时间/D7留存 | 胜率分布 |

### 4.4 四大陷阱

**窥探效应（Peeking）**：提前看数据并据此决策，假阳性率从5%膨胀到20%+。对策：预定结束日期，不提前终止；或使用Sequential Testing。

**辛普森悖论（Simpson's Paradox）**：子群体趋势一致，合并后反转。对策：分维度查看结果，使用分层随机化。

**新奇效应（Novelty Effect）**：新功能的新鲜感导致短期指标虚高。对策：测试至少2周；分新/老用户分析；观察指标时间趋势。

**多重比较**：多变体/多指标膨胀假阳性。对策：Bonferroni校正（α/k）；区分主指标和探索性指标。

## 五、LTV预测与变现优化

### 5.1 LTV预测模型选型

| 数据成熟度 | 推荐模型 | 所需数据 | 精度 |
|-----------|---------|---------|------|
| 刚上线（D0-D7） | 留存外推法 | D1-D7留存 + ARPDAU | 中 |
| 有2周+数据 | ARPU拟合法 | D1-D14累计ARPU | 中-高 |
| 有30天+历史数据 | 用户分群法 | 多维度Cohort数据 | 高 |
| 大数据量+技术团队 | ML预测法 | D0-D3行为特征 | 极高（95%） |

### 5.2 留存外推法公式

```
Retention(d) = a × d^b          // 幂函数拟合
Lifetime = Σ Retention(d)        // 留存曲线下面积
LTV = Lifetime × ARPDAU          // 生命周期 × 日均收入
```

### 5.3 K系数快速预测法

```
K = D90 LTV / D3 LTV（基于历史Cohort计算）
新Cohort预测D90 LTV = 新Cohort D3 LTV × K
```

### 5.4 广告变现优化

**Waterfall vs Bidding**：2025年In-App Bidding已成主流，可带来10-15%的eCPM提升。新项目直接用Bidding。

**频次vs留存权衡**：
- Interstitial每30-45分钟不超过1次
- Rewarded Video无硬上限（用户主动选择）
- 新用户前3天降低广告频次保护留存
- A/B测试不同频次方案，护栏指标=D7留存

### 5.5 混合变现LTV

```
Total LTV = IAP LTV + Ad LTV
Ad LTV = Σ(Daily Impressions × eCPM / 1000)
```

休闲游戏70%+收入可能来自广告。只算IAP的ROAS会严重低估实际盈利能力。必须IAP+广告合并计算ROAS。

### 5.6 首充优化

- 价格：$0.99-$2.99（低门槛）
- 时机：D1-D3，用户遇到第一个"卡点"时
- 包装：标注"10x价值"或"限时85%折扣"
- 限时：24-48小时倒计时
- 目标：付费率从2%提升到5-8%

## 六、多市场对比分析

### 6.1 对比框架

```
Step 1: 建立基准市场 → 统一Cohort口径 → 统一货币USD
Step 2: 标准化对比（留存/变现/获客/参与/K值/技术）
Step 3: 归因差异（市场特性/产品适配/运营执行）
Step 4: 按ROI潜力排序资源
```

### 6.2 Go/Scale/Optimize/Kill决策

| 留存 | ROAS达标 | ROAS不达标 |
|------|---------|-----------|
| 达标（D1>30%且D7>10%） | **SCALE**（放量） | **OPTIMIZE**（优化变现/CPI） |
| 不达标 | **OPTIMIZE**（先修产品） | **KILL/HOLD**（放弃或大改） |

**量化门槛**：
- LTV/CPI > 1.5 → 积极放量
- LTV/CPI 0.8-1.2 → 优化中
- LTV/CPI < 0.8（无优化空间） → 放弃

### 6.3 各区域数据特征

| 市场 | ARPU水平 | CPI水平 | eCPM水平 | 核心特征 |
|------|---------|---------|---------|---------|
| 北美 | 极高（~$60/年） | 极高 | 全球最高 | iOS/Android均衡 |
| 日韩 | 极高 | 极高 | 仅次欧美 | 日本iOS主导65%+ |
| 西欧 | 高 | 高 | 增长强劲 | 多语言挑战/GDPR |
| 中东 | 中-高（海湾接近欧美） | 中 | 中 | 增速最快+18%/斋月关键 |
| 东南亚 | 低 | 极低 | T3水平 | Android 80%+/低端设备适配 |
| 拉美 | 低-中 | 低 | T3水平 | K-Factor潜力大 |

## 七、核心框架总览

### 框架一：数据诊断三步法
接收问题 → 定位指标偏离（与基准对比） → 按诊断决策树逐层排查 → 输出原因+行动方案。数据分析的价值不在于看数字，而在于回答"为什么"和"怎么办"。

### 框架二：留存诊断分层体系
D1看FTUE（30秒正反馈+教程优化+技术稳定+渠道匹配） → D7看内容深度（2周+内容+进阶系统+社交绑定+每日激励） → D30看Meta层（叙事/收集/Battle Pass+LiveOps节奏+经济平衡）。每层诊断独立，对症下药。

### 框架三：A/B测试六步标准流程
假设→指标→分组→样本量→时长→分析。关键约束：MDE 2-5%，α=0.05，Power=80%，时长至少2周。提前确定结束条件，不窥探不提前终止。

### 框架四：LTV预测阶梯模型
按数据成熟度选择：D7数据用留存外推法 → D14+用ARPU拟合法 → D30+用分群法 → 大数据量用ML。每个阶段的模型精度和数据需求都不同，不必一步到位。

### 框架五：广告变现三维优化
eCPM优化（格式选型+网络竞价+Bidding替代Waterfall） × 频次优化（A/B测试找留存/收入平衡点） × 用户分层（Whale减广告推IAP，Non-payer最大化广告）。三个维度独立调优，交叉验证。

### 框架六：多市场Go/Scale/Optimize/Kill矩阵
留存×ROAS两维四象限决策。SCALE=双达标放量，OPTIMIZE=单达标优化短板，KILL=双不达标退出。用LTV/CPI比值做量化判定，>1.5放量，<0.8放弃。

### 框架七：埋点三层+核心路径法
页面级→操作级→业务级三层架构 × 注册→新手→首局→首充→留存→流失六阶段核心路径。服务端保证经济数据权威性，客户端捕获交互行为，关键转化双端验证。

## 八、决策启发式

### 1. "留存是因，变现是果"
留存差的时候优化变现是饮鸩止渴。D1留存每提升5%，D30累计DAU可能增加15-20%，LTV自然水涨船高。先把留存做到品类基准以上，再谈变现优化。

### 2. "D1留存35%是休闲游戏的生死线"
D1低于35%的休闲游戏不值得大规模买量。在FTUE没有做好之前投放的每一分钱，都在购买"看一眼就走"的用户。D1<25%时应全部精力放在产品优化上。

### 3. "不要问'数据是什么'，要问'数据说明什么'"
DAU下降10%不是结论，而是问题的起点。需要拆解：是新增下降还是留存下降？如果是留存，是D1还是D7？如果是D7，是全渠道还是某个渠道？层层追问才能找到根因。

### 4. "埋点是分析的地基，地基歪了楼一定歪"
上线前花1周设计好埋点方案，好过上线后花3个月补数据。核心路径埋点（注册→首局→首充→留存→流失）是最低要求。没有数据的直觉判断和有数据的错误判断一样危险。

### 5. "A/B测试的最大敌人是提前下结论"
跑了3天看到p=0.03就宣布胜利，是在自欺欺人。窥探效应可以让假阳性率从5%膨胀到20%+。预先定好结束条件，到期再看结果，是纪律而非选择。

### 6. "只算IAP不算广告的LTV是自欺欺人"
休闲游戏70%+收入来自广告。如果只看IAP ROAS，会错误暂停很多实际盈利的Campaign。必须IAP+广告合并计算Total LTV和混合ROAS。

### 7. "同一个游戏在不同国家可能是完全不同的产品"
美国D1留存40%的游戏在东南亚可能只有25%，不是因为用户差，而是设备性能、网络环境、文化偏好完全不同。不能用一个市场的数据预期另一个市场的表现。

### 8. "棋牌游戏的胜率分布是产品健康的晴雨表"
胜率分布严重偏斜（少数人赢走大部分筹码）意味着匹配系统或Bot策略有问题。新手被碾压会快速流失，高手无挑战也会离开。理想状态是接近正态分布，中心在45-55%。

### 9. "先在便宜的市场犯错，再在贵的市场放量"
东南亚/拉美的CPI可能只有美国的1/10。在低成本市场验证埋点方案、A/B测试流程、留存优化手段，再带着成熟的方法论进入美国/日本。反过来做是用最贵的流量做实验。

### 10. "LTV/CPI>1.5才值得放量，<0.8就该止损"
这个比值是投放决策的核心指标。1.0是盈亏平衡点（还没算运营成本），1.5以上才有足够安全边际。低于0.8且经过2轮优化无改善，果断放弃比继续烧钱更理智。
