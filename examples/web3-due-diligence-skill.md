# Web3 项目尽调框架

> 在加密市场，你不做尽调，你就是别人的退出流动性。

## 使用说明

**这个Skill不扮演角色，是一个工具。** 用户说"帮我尽调XXX项目"时，按照下面的工作流自动执行。

---

## 🚫 快速否决清单（Instant Kill List）

**在做任何深入分析之前，先过这8条。出现任何一条，直接放弃，不用继续看。**

| # | 红旗 | 为什么是即死条件 |
|---|------|----------------|
| 1 | 合约未开源且未审计 | 无法验证代码，可能藏后门 |
| 2 | 流动性未锁定或锁定期<30天 | 团队可随时撤走流动性（经典rug pull） |
| 3 | 合约含无限铸币函数且无治理约束 | Owner可无限增发抛售 |
| 4 | 蜜罐合约——只能买不能卖 | 隐藏sell限制，确认的犯罪模式 |
| 5 | 创始团队完全匿名且无任何可验证产品 | 匿名+无产品=rug pull最强预测指标 |
| 6 | 单一地址持有>50%代币供应量 | 极端集中持仓，可单方面崩盘 |
| 7 | 项目已上Rekt.news排行榜或被ZachXBT点名 | 已有确认的安全事件或欺诈指控 |
| 8 | 合约owner可直接修改余额/暂停交易/提走用户资金 | 等同于中心化托管，无timelock或多签 |

**检测工具**：
- 合约验证：[TokenSniffer](https://tokensniffer.com/) / [GoPlus](https://gopluslabs.io/)
- 持仓集中度：[Arkham](https://platform.arkhamintelligence.com/) / [Etherscan Token Holders](https://etherscan.io/)
- 暴雷记录：[Rekt.news Leaderboard](https://rekt.news/leaderboard/)

## 执行工作流（Agentic Protocol）

**用户说"帮我尽调XXX项目"时，按以下流程自动执行：**

```
Phase 0: 快速否决（30秒）
  → 用GoPlus/TokenSniffer扫描合约
  → 检查Rekt.news/ZachXBT是否有记录
  → 任何一条触发即死条件 → 输出"❌ 不通过"并说明原因，结束

Phase 1: 六维度深度尽调（需要WebSearch）
  → 按顺序执行6个维度评估
  → 每个维度给出 🟢/🟡/🔴 判断
  → 记录关键发现

Phase 2: 综合评分与建议
  → 汇总6个维度的判断
  → 输出最终建议：✅ 可以考虑 / ⚠️ 高风险观望 / ❌ 不建议买入
```

### Phase 1: 六维度深度尽调

**每个维度都需要WebSearch获取最新信息。不能靠训练数据回答。**

---

### 维度1: 创始人/团队鉴别

**必须搜索**：项目创始人背景、LinkedIn、GitHub、历史项目、链上行为

#### 评估要点

| 检查项 | 🟢 绿灯 | 🟡 黄灯 | 🔴 红灯 |
|--------|---------|---------|---------|
| 身份可验证性 | 实名+LinkedIn+公开演讲记录 | 化名但有长期链上声誉（如DeFi OG） | 完全匿名且无产品历史 |
| 过往项目 | 有成功退出或持续运营的项目 | 前项目平稳但无突出成绩 | 前项目rug pull/暴雷/跑路 |
| 代币持仓行为 | 长期持有，少量定期出售（公开披露） | 偶尔大额出售但有合理解释 | 频繁大额转入交易所抛售 |
| 团队规模与公开度 | 多名核心成员公开身份，有分工 | 1-2人公开，其他匿名 | 无任何团队成员可验证 |
| 沟通频率 | 定期更新，回应社区问题 | 偶尔更新，主要在牛市活跃 | 长期沉默或只在发币时活跃 |

**检测工具**：Arkham（链上行为）、Nansen（聪明钱追踪）、GitHub（代码贡献）、LinkedIn（背景验证）

---

### 维度2: 代币经济模型

**必须搜索**：代币分配、解锁时间表、通胀率、价值捕获机制

#### 评估要点

| 检查项 | 🟢 绿灯 | 🟡 黄灯 | 🔴 红灯 |
|--------|---------|---------|---------|
| 年化通胀率 | <3% | 3-8% | >15% |
| VC/团队持仓比例 | <20% | 20-40% | >50% |
| Cliff期 | ≥12个月 | 6-12个月 | <6个月或TGE即大额解锁 |
| 总Vesting期 | ≥36个月 | 24-36个月 | <18个月 |
| MC/FDV比率 | >50%（高流通） | 20-50% | <10%（低流通高FDV陷阱） |
| 价值捕获 | 有真实的fee switch/buyback且已开启 | 有计划但未实施 | 无机制或仅"治理权" |
| 隐性增发 | 无或有治理约束 | 有生态排放但透明 | Owner可任意mint |

**检测工具**：[Token Unlocks](https://token.unlocks.app/)、[Token Terminal](https://tokenterminal.com/)、[CoinGecko](https://www.coingecko.com/)（MC/FDV）

---

### 维度3: 链上数据真实性

**必须搜索**：TVL、日活用户、协议收入、交易量

#### 评估要点

| 检查项 | 🟢 绿灯 | 🟡 黄灯 | 🔴 红灯 |
|--------|---------|---------|---------|
| TVL来源 | 有机增长，无积分激励或激励已结束 | 有积分激励但用户留存率>30% | 激励驱动，激励停止后TVL暴跌 |
| DAU/MAU粘性 | >20%（DAU/MAU比） | 10-20% | <5%（几乎无回头用户） |
| 协议收入 | 来自手续费/利息等可持续来源 | 部分来自代币激励 | 完全依赖代币激励（印钱补贴） |
| Revenue/TVL | DEX 5-20%年化、借贷1-5%年化 | 低于行业平均但稳定 | 远低于平均或为负 |
| 交易量真实性 | 多交易所分布，模式自然 | 集中在1-2个交易所 | 明显wash trading迹象 |

**检测工具**：[DeFiLlama](https://defillama.com/)（TVL/Revenue）、[Dune Analytics](https://dune.com/)（自定义查询）、[Nansen](https://nansen.ai/)（钱包标签）

---

### 维度4: 融资与估值

**必须搜索**：融资轮次、投资方、估值、做市商

#### 评估要点

| 检查项 | 🟢 绿灯 | 🟡 黄灯 | 🔴 红灯 |
|--------|---------|---------|---------|
| 投资方质量 | Tier 1（a16z/Paradigm/Polychain） | 知名但非顶级基金 | 不知名基金或无法验证 |
| 估值合理性（P/S） | <20x年化收入 | 20-50x | >100x或无收入 |
| FDV/TVL | <3x | 3-10x | >20x |
| 做市商 | 知名做市商（Wintermute/GSR）+透明协议 | 有做市商但协议不透明 | DWF Labs或无做市商 |
| OTC折扣 | 无大规模OTC转让记录 | 有但折扣<30% | 大量OTC以>50%折扣转让 |

**检测工具**：[Crunchbase](https://www.crunchbase.com/)（融资记录）、[Messari](https://messari.io/)（估值数据）、[Token Terminal](https://tokenterminal.com/)（P/S比）

### 维度5: 叙事周期判断

**必须搜索**：项目所在赛道的热度、竞品数量、媒体关注度

#### 叙事四阶段判断

| 阶段 | 信号 | 投资策略 |
|------|------|---------|
| 🟢 **萌芽期** | GitHub活跃但Star少、少数VC布局、市值<$100M、只有开发者讨论 | 最佳买入窗口，但需承受长期等待 |
| 🟡 **共识期** | 头部VC进入、市值$1B-$10B、行业媒体密集报道、交易所开始上线 | 仍可参与，但需更严格的估值筛选 |
| 🔴 **过热期** | 散户涌入、仿盘爆发、名人推广、Google Trends创新高、FDV远超基本面 | 不要新入场，考虑逐步退出 |
| ⚫ **退潮期** | 项目方跑路/pivot、TVL暴跌、叙事从主流消失、只剩"钻石手"社区 | 等待下一个周期，或寻找幸存者 |

#### 真创新 vs 旧概念换皮

| 检查项 | 真创新 | 旧概念换皮 |
|--------|--------|-----------|
| 技术新颖度 | 引入新的密码学原语/共识机制/经济模型 | Fork现有协议+改名换logo |
| 使用场景 | 解决了之前无法解决的问题 | 解决的问题已有更好的方案 |
| 开发者生态 | 吸引了新的开发者群体 | 主要是空投猎人和投机者 |

### 维度6: 风险红旗扫描

**对照详细红旗清单（`references/research/06-red-flags.md`）逐项检查。**

#### 分类检查

| 类别 | 关键红旗（出现任一条升级为🔴） |
|------|------|
| **合约安全** | 未审计、审计后重大改动、Critical/High未修复、Proxy admin为单一EOA |
| **团队行为** | 频繁换项目、创始人大额抛售、社交媒体过度营销但无产品进展 |
| **代币结构** | 近期大额解锁（>流通量10%/月）、团队分配>40%、无锁仓或已过cliff |
| **社区质量** | Discord/TG中机器人比例>50%、付费KOL推广为主、无真实技术讨论 |
| **技术活跃度** | GitHub 90天无提交、纯fork无原创改进、测试网>12个月未上主网 |

```
💡 建议: ✅ 可以考虑 / ⚠️ 高风险观望 / ❌ 不建议买入
理由: [2-3句话]
━━━━━━━━━━━━━━━━━━━━
```

### 综合判断标准

| 绿灯数 | 红灯数 | 建议 |
|--------|--------|------|
| ≥4 绿灯，0 红灯 | — | ✅ 可以考虑（仍需控制仓位） |
| ≥3 绿灯，≤1 红灯 | 红灯不在团队/合约维度 | ⚠️ 高风险观望（等待红灯维度改善） |
| <3 绿灯 或 ≥2 红灯 | — | ❌ 不建议买入 |
| 团队或合约为红灯 | — | ❌ 不建议买入（无论其他维度多好） |

### 附加规则

- **团队和合约安全是硬性否决维度**——这两个为🔴时，其他维度全绿也不买
- **叙事处于过热期时自动降级**——所有🟢降为🟡
- **近30天内有大额解锁时加注意事项**

## 执行注意事项

### 必须使用WebSearch的场景
- 查询项目最新TVL/收入/用户数
- 查询创始人背景和历史
- 查询最近的融资和解锁事件
- 查询合约审计状态
- 查询Rekt.news/ZachXBT是否有相关记录

### 不需要搜索的场景
- 用户问"什么是二次方融资"这类概念性问题
- 用户问框架本身的使用方法

### 输出风格
- **简洁直接**——不是写研报，是给散户做决策
- **用表格**——方便快速扫描
- **明确结论**——不说"看个人风险偏好"，给出明确的✅/⚠️/❌
- **标注不确定性**——如果某个维度信息不足，标为"⬜ 信息不足，无法判断"

## 诚实边界

此框架的局限：

- **信息滞后**：链上数据和公开信息总是滞后于内部人。团队可能已经在你看到数据前采取了行动
- **无法检测所有骗局**：精心设计的骗局可以通过所有表面检查（FTX就通过了大部分传统尽调）
- **叙事判断主观**：四阶段模型是框架不是真理，阶段判断存在主观性
- **工具依赖**：DeFiLlama、Arkham等工具的数据本身可能不完整或有延迟
- **不能替代专业审计**：这是散户级尽调，不能替代专业智能合约审计或机构级尽调
- **市场不可预测**：即使一个项目通过了所有检查，也可能因为系统性风险（如整个市场崩盘）而亏损

**最重要的一条：通过尽调不等于一定赚钱。尽调的目的是排除明显的烂项目，不是保证找到好项目。**
