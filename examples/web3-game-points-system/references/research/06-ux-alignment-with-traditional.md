# 06 与传统欢乐豆的体验对齐 + Web3独特价值

## 核心命题

用户层面应该"感觉跟QQ欢乐豆一模一样"——登录送豆、打牌赢豆输豆、买道具花豆。但底层Web3技术应该带来传统方案做不到的增量价值。

---

## QQ欢乐豆经济系统复盘

### 核心机制

| 机制 | 细节 |
|------|------|
| 日工资 | 余额低于1000时，每日自动发放4次×1000 |
| 入场费 | 不同级别牌桌（初级场/中级场/高级场）不同入场费 |
| 对局结算 | 赢家获得输家的积分（零和+系统抽水） |
| 道具商城 | 记牌器/加倍卡/表情包消耗积分 |
| 充值通道 | Q币→欢乐豆，明确法币价格 |
| 不可提现 | 积分不可兑回法币 |
| 等级系统 | 积分余额决定段位和可进入的牌桌 |

### 用户心理模型

```
用户认知：
- 欢乐豆 = "玩游戏的筹码"
- 输完了 = "等明天日工资" 或 "花几块钱充一点"
- 赢很多 = "成就感 + 可以打更高级场"
- 从不想"欢乐豆值多少钱"（因为不可变现）

关键体验节点：
1. 注册 → 立刻有豆，立刻能玩
2. 输完了 → 不焦虑（明天又有）
3. 赢很多 → 炫耀 + 升级
4. 想要道具 → 花豆（不心疼，因为豆"不值钱"）
```

## 体验对齐清单

### 必须保持一致的体验

| 传统欢乐豆体验 | 链上欢乐豆实现 | 用户感知差异 |
|--------------|--------------|------------|
| 注册送豆 | gameMint(newUser, 10000) | 无差异 |
| 日工资 | 定时任务触发gameMint | 无差异 |
| 打牌输赢 | gameBurn(loser) + gameMint(winner) | 无差异 |
| 买道具 | gameBurn(buyer, price) | 无差异 |
| 充值 | 法币支付 → gameMint | 无差异 |
| 余额查询 | balanceOf(address) | 无差异（数字展示完全相同） |
| 牌桌等级 | 链下逻辑根据balance分流 | 无差异 |

### 绝不暴露的Web3细节

- 钱包地址（用户名/手机号替代）
- Gas费（Paymaster全量代付）
- 区块确认等待（链下缓存即时响应）
- 交易Hash（除非用户主动查看"链上记录"）
- 链名/网络切换
- 助记词/私钥

## Web3带来的独特增量价值

### 价值1: 可验证的公平性

```
传统欢乐豆：
- 用户质疑"系统是不是故意让我输？"
- 腾讯说"没有"→ 但无法证明

链上欢乐豆：
- 每局结果的Merkle Root上链
- 随机数种子可验证（VRF）
- 用户可独立验证"系统没有作弊"
- 在"我的记录"页面增加"查看链上凭证"按钮（可选功能）
```

### 价值2: 真正的数据所有权

```
传统欢乐豆：
- 腾讯封号 → 积分清零，无追索权
- 服务器关闭 → 数据消失

链上欢乐豆：
- 积分余额在链上 → 即使游戏关服，数据仍在
- 封号只能限制登录，无法清除链上余额
- 用户可导出完整游戏历史
```

### 价值3: 跨平台便携性

```
传统欢乐豆：
- 只在QQ游戏生态内使用
- 换平台 = 从零开始

链上欢乐豆：
- 积分跟随钱包地址（绑定到用户身份）
- 理论上任何授权的游戏都可以读取balance
- 用户的"游戏资历"可跨平台携带
```

### 价值4: 社区驱动的经济参数

```
传统欢乐豆：
- 腾讯单方面决定发放量、消耗率、充值价格
- 用户无话语权

链上欢乐豆（进阶玩法）：
- 活跃用户组成DAO投票决定：
  · 日工资发放量
  · 新活动的奖励额度
  · 新道具的定价
- 治理权基于活跃度（非持仓量），避免金权政治
```

### 价值5: 防灰色交易

```
传统欢乐豆的问题：
- "豆商"在淘宝/闲鱼低价卖豆
- 外挂刷豆 → 灰色产业链
- 腾讯疲于打击

链上欢乐豆解决方案：
- 合约级禁止P2P转账 → 从技术上消灭豆商
- 所有mint操作可追溯 → 外挂刷豆立刻可查
- 链上透明性使灰色交易无处藏身
```

## 渐进式Web3化路线图

### Phase 1: 隐形上链（用户无感）
- 传统登录 + 后台MPC钱包
- 积分合约部署但用户不知道
- 与传统体验100%一致
- 目标：验证技术可行性，用户无流失

### Phase 2: 轻度感知（可选探索）
- 新增"链上记录"入口（可选查看）
- 推出"公平性验证"功能
- 引入链上成就徽章（Soulbound NFT）
- 目标：让好奇用户感受Web3价值

### Phase 3: 跨游戏互通
- 第二款游戏上线，共享积分合约
- 跨游戏成就系统
- "游戏资历"概念建立
- 目标：体现Web3独特优势

### Phase 4: 社区治理（高阶）
- 活跃用户投票权
- 经济参数链上治理
- 开发者生态开放（第三方游戏接入积分系统）
- 目标：建立去中心化游戏平台
