# 即开票产品设计操作系统

> 即开票的核心是一道数学题：在RTP、中奖概率、头奖吸引力三者之间找到让玩家"觉得能赢"同时运营商"确定能赚"的平衡点。
> 本系统覆盖线下实体票和线上e-Scratch两条路径，所有参数均提供西非FCFA市场适配。

## 一、奖级结构设计方法论

### 1.1 核心参数及其关系

即开票设计的五个核心参数相互联动：

```
┌─────────────────────────────────────────────────────┐
│  RTP（返奖率）= 总奖金 / 总票面值                      │
│  Overall Odds（整体中奖率）= 总票数 / 中奖票数          │
│  Prize Tiers（奖级数）= 通常5-10级                     │
│  Top Prize Multiplier（头奖倍数）= 头奖 / 面值          │
│  Print Run（印刷量）= 决定头奖概率的分母                 │
│                                                       │
│  关键公式：                                            │
│  RTP = Σ(每级奖金 × 该级中奖票数) / (面值 × 总票数)     │
│  头奖概率 = 头奖数量 / 总票数                           │
│  整体中奖率 = 中奖票数 / 总票数                         │
└─────────────────────────────────────────────────────┘
```

**行业标准参数范围：**

| 参数 | 行业标准 | 西非建议 | 说明 |
|------|---------|---------|------|
| RTP（线下） | 55%-75% | 58%-68% | 西非运营成本较高（物流、零售网络），RTP空间略小 |
| RTP（线上） | 80%-96% | 75%-85% | 省去印刷物流成本，但需投入技术和推广 |
| 整体中奖率 | 1:3 到 1:5 | 1:3.5 到 1:4.5 | 西非市场偏好"感觉能中"，建议偏高频 |
| 奖级数量 | 5-10级 | 5-7级 | 低面值票（100-200 FCFA）用5级即可，简洁 |
| 头奖倍数 | 5,000x-100,000x | 500x-2,000x | 西非面值低，头奖绝对值比倍数更重要 |
| 底层奖占比 | 90-99%中奖票 | 92-97% | 底层奖 = 面值返还或小额奖 |

**参数联动规则：**
1. RTP上升 → 运营利润下降 → 必须靠量补（或削减头奖数量）
2. 整体中奖率提高 → 每张中奖票的平均奖金下降 → 头奖必须减少
3. 头奖倍数增加 → 头奖概率必须极低（否则RTP爆表）
4. 印刷量增加 → 头奖概率自动稀释 → 可支撑更大头奖

### 1.2 奖级设计心理学

**"近赢"效应（Near-miss Effect）：**
- 设计时让部分非中奖票"差一点就中"（如3个符号需匹配，显示2个匹配）
- 触发"下次一定"心理，提升复购
- 注意：部分司法管辖区对近赢设计有监管限制

**奖级梯度的心理锚定：**
```
最佳实践：奖级间隔应遵循"可感知跳跃"

面值100 FCFA的票 参考设计：
  第1级：100 FCFA（面值返还）← 最高频，让玩家"没亏"
  第2级：200 FCFA（2x）← 小确幸
  第3级：500 FCFA（5x）← 明显收获感
  第4级：2,000 FCFA（20x）← 惊喜感
  第5级：10,000 FCFA（100x）← 口碑传播阈值
  第6级：100,000 FCFA（1000x）← 头奖，改变生活
```

**"保本票"策略：**
- 面值返还（breakeven）票应占所有中奖票的50-70%
- 让整体中奖率数字好看（如"每3张中1张"），大部分是保本
- 实际"赚到钱"的概率远低于整体中奖率

### 1.3 西非市场面值策略

**塞内加尔经济背景：**
- 人均GDP：约$1,760-$1,921/年（2024-2025）
- 日均收入：约2,900-3,200 FCFA（$4.8-$5.3）
- 汇率：655.957 FCFA = 1 EUR（固定挂钩）

**面值矩阵：**

| 面值 | 占日均收入 | 定位 | 头奖建议 | 头奖倍数 | RTP建议 | 购买场景 |
|------|-----------|------|---------|---------|---------|---------|
| 100 FCFA | ~3% | 大众入门 | 100,000 FCFA | 1,000x | 58-62% | 零钱、冲动、日常 |
| 200 FCFA | ~6% | 轻度参与 | 200,000 FCFA | 1,000x | 60-64% | 有意尝试 |
| 500 FCFA | ~16% | 中端主力 | 1,000,000 FCFA | 2,000x | 62-66% | 周末、发薪日 |
| 1,000 FCFA | ~32% | 高端体验 | 5,000,000 FCFA | 5,000x | 64-68% | 特殊场合、节日 |

**LONASE Dare-Dare参考：**
- 品牌名"Dare-Dare"覆盖LONASE全部即开票产品线
- 与LudWin和FDJ Gaming Solutions合作开发
- 已推出"Numero Porte Bonheur"：100 FCFA面值、头奖100,000 FCFA（1,000x）
- 设计洞察：基于当地"幸运数字信仰"和"颜色偏好"的文化适配
- 已推出"But"（足球主题）：利用足球在塞内加尔的极高热度

**西非面值设计原则：**
1. **入门面值必须 <= 日均收入的5%**（100-150 FCFA），确保冲动购买无心理门槛
2. **头奖绝对值比倍数更重要**——"赢100,000 FCFA"比"赢1000倍"更有传播力
3. **FCFA无小数**——所有奖金必须是整数，且建议为100的整数倍（流通方便）
4. **文化适配**——融入当地信仰元素（幸运数字、颜色象征、足球、音乐）
5. **跨UEMOA通用性**——同一面值体系可在8个FCFA国家复用

## 二、线下实体即开票

### 2.1 印刷安全技术

即开票安全是多层防线体系。以下按安全层级从基础到高级排列：

| 安全技术 | 功能 | 规格要点 | 成本等级 | 西非建议 |
|---------|------|---------|---------|---------|
| **乳胶覆盖层** | 遮挡中奖信息 | 碳黑/铝膏 + 丙烯酸树脂；溶剂MEK | 基础（必须） | 所有面值必备 |
| **序列号加密** | 追溯和验证 | 十六进制编码，可分区/分级前缀 | 基础（必须） | 所有面值必备 |
| **条码 + 安全码** | 双重零售验证 | 条码扫描 + 3-4位隐藏安全码手动输入 | 基础（必须） | 所有面值必备 |
| **黑线叠印** | 防复印/扫描 | 覆盖在中奖区域上方的图案线 | 低 | 100-200 FCFA可选 |
| **UV安全油墨** | 隐形验证标记 | 可见光透明，紫外灯下荧光 | 中 | 500+ FCFA建议 |
| **IR安全油墨** | 红外验证 | 红外检测设备识别 | 中 | 500+ FCFA建议 |
| **微文字** | 防扫描复制 | 极小文字，肉眼难辨，复印失真 | 中 | 500+ FCFA建议 |
| **Void if Removed** | 防篡改 | 揭起后显示VOID字样 | 中 | 高面值票必须 |
| **SecurTag涂层** | 零售端认证 | Scientific Games专利；肉眼不可见，专用设备识别 | 高 | 1,000 FCFA建议 |
| **全息标识** | 防伪终极手段 | 3D光学图像，无法用已知印刷技术复制 | 高 | 1,000 FCFA或特殊版 |
| **HD Games印刷** | 高清防伪 | 高DPI复杂印刷，Dimension全息效果 | 高 | 高端系列可选 |

**防入侵测试（合格标准）：**
- 防透光：强光照射不可读取下方信息
- 防物理：物理撬开不可保持完整性
- 防化学：常见溶剂不能溶解覆盖层而不损坏信息
- 防冷冻/加热：极端温度不影响覆盖层完整性
- 防X射线：X射线不可成像下方信息

### 2.2 生产与物流安全链

**14步生产流程（行业标准）：**
1. 游戏设计和编程
2. 随机算法生成中奖数据
3. 数据加密
4. 纸stock准备（大卷，最重2000磅/卷）
5. 基础层印刷（CMYK + PMS色）
6. 中奖信息印刷（加密数据）
7. 安全涂层施加
8. 乳胶覆盖层印刷
9. 外观图案/品牌印刷
10. 冷箔/全息效果（如适用）
11. 切割和打包
12. 质量抽检
13. 安全运输（密封包装，GPS追踪）
14. 零售终端激活

**WLA-SCS:2020安全标准：**
- 多层重叠安全系统（防火墙、网络安全）
- 游戏数据完全加密
- 生产到分销全链路审计追踪
- 参考：世界彩票协会安全控制标准

**西非物流特殊考量：**
- 高温高湿环境：覆盖层材料需耐热耐湿（乳胶配方调整）
- 分销网络分散：GPS追踪更关键
- 零售端设备简陋：条码+手动安全码的双重验证最可靠（不依赖高端设备）
- 假币环境：防伪层级需高于成熟市场同面值产品

### 2.3 零售端设计

**陈列和销售：**
- 票面设计需在简陋店面环境中视觉突出
- 颜色策略：明亮色彩（红、金、绿）在西非市场更具吸引力
- 游戏规则必须简单——一句话能说清怎么玩
- 多语言：法语（官方语）+ 沃洛夫语（通用语）图标化设计

**零售商培训：**
- 条码扫描 + 安全码输入操作
- 假票识别（UV灯检查，如配备）
- 中奖验证和小额兑付（低级奖零售端即兑）
- 大额奖引导到LONASE中心

## 三、线上即开游戏（e-Scratch）

### 3.1 服务器端预判定架构

线上即开票的核心问题：**结果在哪里生成？**

```
┌──────────────────────────────────────────────────┐
│  方案A：实时RNG判定（推荐起步方案）                  │
│  ──────────────────────────                       │
│  玩家点击"购票" → 服务器RNG生成结果 →               │
│  结果写入数据库 → 前端播放刮开动画 →                 │
│  动画结束显示结果                                   │
│                                                    │
│  优势：简单、灵活、无"虚拟库存"管理                   │
│  劣势：需严格RNG认证                                │
├──────────────────────────────────────────────────┤
│  方案B：奖池预判定 + 随机抽取                        │
│  ──────────────────────────────                   │
│  预生成N张虚拟票（含中奖/未中奖）放入奖池 →           │
│  玩家点击"购票" → 服务器从池中随机抽取一张 →          │
│  前端播放动画 → 显示结果                             │
│                                                    │
│  子类型B1 - Depleting Pool（消耗型）：               │
│    票用完即止，类似实体票，可预测收入                   │
│  子类型B2 - Replenishing Pool（补充型）：             │
│    自动补充，持续运营，适合长期线上游戏                 │
│                                                    │
│  优势：RTP精确可控、更接近实体票逻辑                   │
│  劣势：需要奖池管理系统                              │
└──────────────────────────────────────────────────┘
```

**架构组件（以Allwyn Apollo RGS为参考）：**

| 组件 | 功能 | 说明 |
|------|------|------|
| Remote Game Server (RGS) | 游戏托管和结果生成 | 核心引擎 |
| RNG Module | 随机数生成 | 需GLI-19认证 |
| Prize Pool Manager | 奖池管理 | Depleting/Replenishing模式 |
| Game Asset Delivery | 动画和UI推送 | 动态交付到玩家设备 |
| Transaction System | 交易记录 | 购票、中奖、兑付全流程 |
| Reporting Engine | 数据报表 | 实时监控RTP、销售、奖金分布 |

**西非e-Scratch适配：**
- 带宽优化：动画资源需极度轻量化（<500KB），支持2G/3G网络
- USSD/SMS集成：不是所有玩家有智能手机，考虑短信即开票
- 移动支付：与Orange Money、Wave等本地移动钱包对接
- 离线容错：网络中断时的交易一致性保障

### 3.2 RNG与公平性保障

**GLI-19标准核心要求：**
- **不可预测性**：产出序列不可被任何方法预测
- **无偏差**：不向任何特定结果倾斜
- **不可重复**：相同输入不产生相同输出序列
- **统计随机性**：通过NIST/Diehard等统计测试套件
- 适用范围：在线游戏平台、远程博彩、虚拟游戏、管理/监控/交易系统

**认证路径：**
1. 选择GLI认证的RNG模块（或自研后送检）
2. 提交至Gaming Laboratories International测试
3. 通过后获得GLI-19合规证书
4. 部分司法管辖区直接接受GLI-19，部分需在此基础上做本地认证
5. 定期复检（通常年度）

**西非认证现实：**
- UEMOA地区目前无统一iGaming认证框架
- LONASE作为监管方可自定标准
- 建议：主动采用GLI-19作为基准，建立领先优势
- 合作方FDJ Gaming Solutions和LudWin本身有成熟认证体系

### 3.3 用户体验要点

**刮开交互设计：**
- 模拟手指刮开的触感反馈（震动）
- 渐进式揭示——让玩家"控制"揭示速度
- 声音设计：刮开声、中奖音效、大奖特殊庆祝动画
- "快速揭示"按钮（部分玩家不喜欢慢慢刮）

**中奖体验：**
- 小额奖：简洁提示 + 自动入账
- 中额奖：庆祝动画 + 分享按钮
- 大额奖：全屏庆祝 + 引导到兑付流程

## 四、透明度与信任建设

### 4.1 UK标杆机制

UK National Lottery是全球即开票透明度的金标准：

| 机制 | 做法 | 效果 |
|------|------|------|
| **票面印刷概率** | 每张票背面印整体中奖概率 | 购买前知情 |
| **Game Procedures公开** | 每款游戏发布PDF文件，含完整奖级表、总票数、各级奖数、RTP | 完全可验证 |
| **头奖剩余数实时公布** | 官网1个工作日内更新头奖剩余 | 玩家可判断"还值不值得买" |
| **游戏编号体系** | 票面条码含4位游戏编号 | 可追溯到具体游戏批次 |
| **退市透明** | 头奖全部认领后公告，设兑奖截止期 | 不卖"已无头奖"的票（未激活库存收回） |
| **Gambling Commission监管** | 独立监管机构，信息可通过信息自由法获取 | 第三方信任背书 |

**UK Game Procedures示例（"Just To Say"游戏）：**
- 面值：低价位
- 总RTP：59.98%
- 总奖金：GBP 8,337,386
- 整体中奖率：1:3.82
- 所有信息在 national-lottery.co.uk/c/files/scratchcards/ 公开

### 4.2 西非市场适配

直接照搬UK模式不现实（识字率、互联网普及率限制），需要分层适配：

**第一层：票面透明（必做，零成本）**
- 票面印刷整体中奖概率（法语 + 图示）
- 票面印刷RTP（"每100 FCFA中有XX FCFA回到玩家手中"）
- 简单图标化：用笑脸/星星数量表示中奖概率等级

**第二层：零售端透明（低成本）**
- 零售店张贴每款游戏的奖级海报
- 大奖中出后在零售点公告（"本店已出XXX FCFA大奖"）
- 中奖者故事传播（在当地社区，口碑极其重要）

**第三层：数字透明（中等投入）**
- LONASE官网公布头奖剩余数
- 短信查询服务：发送游戏编号到指定号码，返回头奖剩余信息
- 社交媒体（Facebook/WhatsApp）定期公布中奖统计

**第四层：制度透明（长期建设）**
- 公开Game Procedures文件（参考UK模式）
- 引入独立审计
- 年度报告公布各游戏实际RTP vs 设计RTP

**信任建设的文化适配：**
- 西非社会信任基于"人际网络"而非"文件公开"
- 中奖者现身说法（视频/社区活动）比PDF文件更有说服力
- 清真寺/教堂/社区领袖背书可能比政府认证更有效
- 广播电台开奖/公告在农村地区覆盖最广

## 五、产品生命周期管理

### 5.0 生命周期全景

```
构思 → 设计 → 审批 → 模拟验证 → 印刷/开发 → 上市 → 运营监控 → 退市 → 兑奖期 → 归档
 │      │      │       │          │        │       │         │       │        │
 1月    2月    3月     3月        4-5月    6月     6-18月    18月    24月     归档
                                                   ↑
                                            持续监控+决策点
```

### 5.3 退市：触发条件和流程

**退市触发条件（任一满足即评估）：**
1. **头奖清零**：所有头奖已被认领 → 必须退市
2. **销售目标达成**：预设销售量已完成
3. **销售疲劳**：连续4周日均销量低于预期的30%
4. **安全事件**：发现伪造票或安全漏洞
5. **商业决策**：为新游戏腾出零售空间
6. **合规要求**：监管机构要求

**退市流程：**
1. 停止向零售商分发新包装
2. 已激活库存可继续销售至售罄
3. 未激活库存全部收回（物理销毁）
4. 官网/零售点公告退市信息
5. 设定兑奖截止期：退市后**180天**（建议，参考美国标准）
6. 截止后未认领奖金按当地法规处理（通常归入公益金）
7. 游戏数据归档（保留至少5年）

### B. 方案审查清单（触发"这个方案合不合理"时使用）

**数学合理性：**
- [ ] RTP在目标范围内？（线下58-68%，线上75-85%）
- [ ] 总奖金 = Σ(各级奖金 × 各级中奖数)，计算无误？
- [ ] 整体中奖率在1:3到1:5之间？
- [ ] 头奖概率与印刷量匹配？（头奖数/总票数）
- [ ] 各级中奖票数之和 + 未中奖票数 = 总票数？
- [ ] 各级奖金总额之和 = 总奖金？
- [ ] RTP = 总奖金 / 总票面值？
- [ ] 面值返还票占中奖票的50-70%？

**商业合理性：**
- [ ] 面值符合目标市场购买力？（<= 日均收入的5%为入门）
- [ ] 头奖绝对值足够有吸引力？（至少为月均收入的1-2倍）
- [ ] 运营商毛利率(1-RTP)覆盖印刷+物流+零售佣金+利润？

- [ ] 序列号加密方案可行？

### D. 诚实边界

6. **面值策略基于宏观经济数据推导**——实际定价需结合LONASE零售网络数据和竞品分析

8. **文化适配建议基于公开信息**——深度本地化需要实地消费者调研
