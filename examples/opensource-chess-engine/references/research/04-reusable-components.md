# 04-可复用组件识别

## 概览

开源棋牌项目中可复用的通用组件，避免重复造轮子。

## 1. 用户系统

### Nakama内置方案（最完善）
- 设备ID登录、邮箱登录、社交账号登录（Google/Facebook/Apple/Steam等）
- 用户资料存储
- 好友系统（添加/删除/屏蔽）
- 群组系统（创建/加入/管理）
- 来源：https://heroiclabs.com/nakama/

### 自建方案关键模块
- JWT Token认证
- OAuth2第三方登录
- 账号绑定（游客->正式账号）
- 多设备登录互踢策略

### 推荐复用
- **Nakama** 用户系统最完善，可直接使用
- 如果不用Nakama，推荐用成熟的认证库（如Go的casbin/casdoor）

## 2. 支付/经济系统

### 已有实现
- **Nakama**：内置IAP验证（Apple App Store、Google Play、Huawei AppGallery）
- **casino-server**：简单的虚拟金币系统（Redis存储）

### 棋牌经济系统核心模块
| 模块 | 说明 | 关键设计 |
|------|------|---------|
| 钱包 | 虚拟货币余额管理 | 原子操作、防超发、流水日志 |
| 充值 | 对接支付渠道 | 回调验签、幂等处理、对账 |
| 提现 | 虚拟货币兑换 | 风控审核、限额、冷却期 |
| 赠送 | 玩家间转账 | 防刷机制、额度限制 |
| 商城 | 道具购买 | 库存管理、限购 |

### 注意事项
- 所有金额操作必须使用**整数**（分为单位），避免浮点精度问题
- 必须有完整的交易流水日志（可审计）
- 生产环境必须对接真实支付SDK，不能自建支付通道

## 3. 排行榜

### Nakama排行榜（推荐复用）
- 支持多种排序规则
- 赛季排行（自动重置）
- 锦标赛系统（可关联到联赛）
- 性能：专门的排行榜缓存层

### Redis自建方案
```
ZADD leaderboard:daily <score> <playerID>
ZREVRANK leaderboard:daily <playerID>      // 获取排名
ZREVRANGE leaderboard:daily 0 99 WITHSCORES // Top 100
```
- 简单高效，适合中小规模
- 需自行处理赛季重置和历史快照

### 排行榜类型
| 类型 | 刷新周期 | 场景 |
|------|---------|------|
| 实时榜 | 实时 | 当前牌局胜率 |
| 日榜 | 每日0点 | 日赢金币榜 |
| 周榜 | 每周一 | 周积分榜 |
| 赛季榜 | 每月/每季 | 段位排行 |
| 总榜 | 永久 | 累计赢局数 |

## 4. 聊天系统

### Nakama聊天（最完善）
- 1对1私聊
- 群组聊天
- 房间内聊天
- 全局频道
- 持久化消息历史

### 棋牌聊天特殊需求
- **快捷短语**：预设表情/语音（"不错"/"快点啊"），避免敏感内容
- **语音消息**：录音上传+播放
- **敏感词过滤**：必须有，涉及合规
- **禁言机制**：管理员/自动检测
- **牌局内聊天**：仅本桌可见

## 5. 数据统计/分析

### 现有框架支持
- **Pitaya**：Prometheus + Statsd + OpenTracing
- **Nakama**：内置Dashboard + 指标收集
- **ioGame**：全链路追踪

### 棋牌核心指标
| 类别 | 指标 |
|------|------|
| 用户 | DAU/MAU、留存率、新增用户、付费率 |
| 游戏 | 场均时长、人均局数、胜率分布、弃牌率 |
| 经济 | ARPU、充值额、提现额、库存金币总量 |
| 技术 | CCU峰值、消息延迟、错误率、服务器负载 |
| 风控 | 异常账号数、作弊检测命中率、投诉率 |

### 推荐技术栈
- 实时指标：Prometheus + Grafana
- 日志采集：ELK（Elasticsearch + Logstash + Kibana）
- 牌局回放：MongoDB存储完整操作序列
- 数仓分析：ClickHouse（大数据量OLAP）

## 6. 后台管理系统

### 现有实现
- **Nakama**：内置Web Dashboard（用户管理、匹配监控、存储浏览）
- **jforgame**：包含后台管理系统
- **ioGame**：内置管理面板

### 棋牌后台核心功能
| 功能 | 说明 |
|------|------|
| 用户管理 | 查看/封禁/解封账号，修改资产 |
| 游戏管理 | 创建/关闭房间，调整参数 |
| 公告系统 | 推送通知、维护公告 |
| 客服工具 | 查看牌局回放、处理投诉 |
| 运营活动 | 签到/任务/充值活动配置 |
| 数据看板 | 收入/在线/留存等核心指标 |
| 风控系统 | 异常行为检测、自动处置 |

## 复用决策矩阵

| 组件 | 推荐复用来源 | 自建难度 | 备注 |
|------|-------------|---------|------|
| 用户系统 | Nakama | 中 | 或用casdoor等开源认证 |
| 排行榜 | Nakama/Redis | 低 | Redis ZSET即可满足 |
| 聊天 | Nakama | 中高 | 自建需考虑消息可靠性 |
| 支付 | 自建+SDK | 高 | 必须对接官方支付渠道 |
| 数据统计 | Prometheus+Grafana | 中 | 开源方案成熟 |
| 后台管理 | 自建（参考Nakama Dashboard） | 中高 | 业务定制性强 |
