# 休闲合成游戏设计方法论

> Merge游戏的本质是一台精密的"满足感制造机"：用合成的即时反馈驱动短期参与，用发现的好奇心驱动中期探索，用收集的完成感驱动长期留存。设计者的工作是校准这三层满足感的产出节奏，同时在恰当的摩擦点嵌入变现触发器。

## 使用规则

| 用户说 | 触发动作 |
|--------|----------|
| 「设计一款合成游戏」 | 输出完整设计方案（核心循环→合成树→进度→经济→变现→留存） |
| 「合成树怎么设计」 | 输出合成树深度/宽度设计框架 + 满足感曲线 |
| 「经济系统怎么平衡」 | 输出双货币体系 + Taps/Sinks平衡模型 |
| 「怎么变现」 | 输出变现模式选择矩阵 + 定价策略 + 广告触发点 |
| 「留存怎么做」 | 输出留存机制全景（签到→事件→社交→推送→回流） |
| 「分析XXX产品」 | 用六维度框架拆解指定Merge游戏 |
| 「能量系统怎么设计」 | 输出能量经济平衡模型 + 会话节奏设计 |

## 一、核心循环设计

### 1.1 四阶闭环

Merge游戏的基础循环是一个四阶闭环：

```
发现(Discover) → 合成(Merge) → 升级(Upgrade) → 收集(Collect)
       ↑                                              |
       └──────────────────────────────────────────────┘
```

- **发现**：通过能量消耗、生成器产出、任务奖励获得物品
- **合成**：将相同物品合并为更高等级物品
- **升级**：获得更高级物品，解锁新合成路径
- **收集**：完成物品链、填充图鉴、达成外围目标

每个阶段都应该产出**即时正反馈**（动画/音效/视觉变化），闭环的完整周期应控制在**30秒-3分钟**内，确保玩家每次会话都能经历多次完整循环。

### 1.2 Merge-2 vs Merge-3 机制选型

| 维度 | Merge-2（合二为一） | Merge-3（合三为一） |
|------|-------------------|-------------------|
| 代表产品 | Merge Mansion, Travel Town | Merge Dragons!, EverMerge |
| 操作门槛 | 低（两个即可合成） | 中（需凑齐三个） |
| 策略深度 | 偏低，快速反馈 | 较高（3合vs5合的掌握感） |
| 空间压力 | 更大（物品快速堆积） | 较小（3→1压缩率高） |
| 满足感节奏 | 频繁小满足 | 间隔大满足 |
| 适合方向 | 大众休闲、低门槛获客 | 中核休闲、策略深度 |

**选型决策规则**：
- 目标用户偏大众/年龄跨度大 → Merge-2
- 追求策略深度和长期LTV → Merge-3
- Merge-3中的5-Merge奖励（凑5个合出2个高级物品而非3合1个）提供"20%进度提升"，创造轻量但满足感强的掌握元素

### 1.3 合成树深度与宽度

**深度**（从最低级到最终物品的层数）：

| 层级段 | 深度 | 设计目标 | 时间预期 |
|--------|------|---------|---------|
| 浅层（1-3级） | 3级 | 单次会话内可达成，给新手即时满足 | 5-10分钟 |
| 中层（4-7级） | 4级 | 日常游戏主体内容 | 1-3天 |
| 深层（8-12级） | 2-4级 | 长期目标锚定 | 1-2周 |

**宽度**（同时活跃的平行合成链数量）：
- 最佳区间：3-5条同时活跃的合成链
- 过宽（>7条）→ 注意力分散、库存压力大
- 过窄（<2条）→ 内容单调、容易疲劳

**合成链类型**（Travel Town的创新实践）：
- **标准链**：A₁ → A₂ → A₃ → ... → A_n
- **Nestled chains**：链中途产生新生成器，再开启子链
- **Combined chains**：两条独立链合并产出最终物品
- **无能量生成器**：冷却后自动在相邻格子产出

### 1.4 发现机制：随机性与确定性的平衡

Merge Dragons的核心竞争力在于"发现"设计哲学——刻意不展示完整合成链，让未知保持神秘。

**配比原则**：

| 机制 | 确定性 | 随机性 | 设计意图 |
|------|--------|--------|----------|
| 合成结果 | 高 | 低 | 可预期的进度感 |
| 生成器产出 | 中 | 中 | 每次点击的期待感 |
| 任务奖励 | 高 | 低 | 进度可控 |
| 事件掉落 | 低 | 高 | 惊喜和稀缺感 |
| 宝箱内容 | 低 | 高 | 变现触发点 |

**底线规则**：核心进度路径保持高确定性（玩家不会因运气差完全停滞），辅助路径引入随机性保持新鲜感。

### 1.5 满足感曲线设计

```
满足感
  ↑
  │        ★ 首次完成完整合成链
  │    ★        
  │  ★   ★         ★ 发现隐藏路径
  │ ★      ★     ★
  │★         ★ ★       （低谷：中间层级磨时间）
  │
  └──────────────────────────→ 游戏进度
```

**四个关键设计点**：
1. **首次合成即高潮**：第一次合成必须产生强烈正反馈（动画+音效+视觉升级）
2. **链式反应奖励**：合成结果恰好触发连锁合成时给予额外奖励
3. **里程碑奖励**：完成一条完整合成链给予特殊奖励，对冲中间层级磨损感
4. **意外发现**：偶尔在合成中产出意外物品，维持新鲜感

## 二、进度系统设计

### 2.1 主线架构选型

| 架构类型 | 特点 | 适合方向 | 代表产品 |
|---------|------|---------|---------|
| **订单驱动型** | 完成角色订单推进主线 | 目标明确，节奏可控 | Travel Town, Merge Mansion |
| **探索驱动型** | 合成解锁新区域/角色 | 探索感强，内容丰富 | Merge Dragons!, EverMerge |

**订单系统设计最佳实践**（Travel Town模式）：
- 同时提供4个活跃订单
- 难度分布：1高难高回报 + 1低难易完成 + 2中等
- 确保玩家始终有可完成目标（避免全部卡住的挫败感）

### 2.2 Meta-Layer类型

| Meta类型 | 机制 | 留存价值 | 代表产品 |
|---------|------|---------|---------|
| 建筑修缮 | 合成工具→修缮区域 | 视觉进度+叙事驱动 | Merge Mansion |
| 城镇建设 | 完成订单→建造建筑 | 收集完成感 | Travel Town |
| 角色收集 | 合成链→解锁角色 | 收藏欲望 | EverMerge |
| 地图解锁 | 治愈土地→扩展世界 | 探索欲望 | Merge Dragons! |

**Meta层设计原则**：
1. 每一步努力都产生可见的环境变化
2. 同时提供2-3条外围目标线，避免单线卡死
3. 前几个目标5-10分钟内可完成，后续逐步拉长

### 2.3 游戏板架构

| 板型 | 特点 | 变现空间 | 代表产品 |
|------|------|---------|---------|
| **扩展式** | 随进度扩展地图 | 高（付费解锁区域） | Merge Dragons!, EverMerge |
| **固定式** | 空间固定不变 | 中（空间管理为策略核心） | Merge Mansion, Travel Town |

扩展式游戏板的物品堆积问题会自然驱动更多变现（清理空间、加速合成）。固定式游戏板则通过空间管理创造策略深度。

### 2.4 能量系统平衡

**能量的三重角色**：
1. 会话节奏控制（限制单次游戏5-20分钟）
2. 变现触发点（能量耗尽是最自然的付费时机）
3. 内容消耗节流阀（防止玩家过快消耗内容）

**能量经济设计**：

```
能量来源（Taps）：             能量消耗（Sinks）：
├── 时间恢复（基础）            ├── 生成器点击
├── 任务奖励                   ├── 特殊合成
├── 事件奖励                   └── 加速模式（如Boost x2）
├── 激励视频（4-6次/天）
├── 好友赠送
└── IAP购买
```

**行业参考**：
- 能量占休闲游戏收入约三分之一
- 设计目标：玩家每4-8小时返回一次
- 会话时长目标：5-20分钟

## 三、经济系统设计

### 3.1 双货币体系

| 货币类型 | 获取 | 消耗 | 角色 |
|---------|------|------|------|
| **软货币** | 合成产出、任务奖励 | 基础购买、解锁区域 | 进度润滑剂 |
| **硬货币** | 少量游戏奖励、IAP | 跳过计时器、购买稀有物品、能量 | 核心变现工具 |

**关键设计模式——"先宠溺再变现"**（Merge Mansion）：
- 新手阶段赠送充裕硬货币（约100颗）
- 体验过高级游戏后，资源急剧稀缺
- 自然产生付费需求

### 3.2 Taps & Sinks平衡模型

```
资源产出（Taps）               资源消耗（Sinks）
┌─────────────────┐           ┌─────────────────┐
│ 生成器产出        │           │ 合成消耗         │
│ 任务/订单奖励     │           │ 建造/解锁消耗     │
│ 事件奖励         │   ━━━►    │ 商店购买          │
│ 每日登录奖励      │           │ 加速/跳过计时器    │
│ 广告奖励         │           │ 能量购买          │
│ IAP购买          │           │ 空间扩展          │
└─────────────────┘           └─────────────────┘
```

**平衡五步法**：
1. 定义核心资源角色
2. 绘制完整资源流向图
3. 基于目标会话时长反推每分钟资源产出量
4. 设计递增消耗梯度（前期丰裕→中后期紧张）
5. 通过A/B测试持续微调

### 3.3 通胀控制

**反通胀核心公式**：

```
资源产出增速 < 资源消耗增速 × 1.1
```

消耗的增长速度应略快于产出（约10%的稀缺压力），既不至于卡死，又保持付费动力。

**主要反通胀机制**：
- 空间限制（游戏板面积固定）
- 递增消耗（后续内容需要更多资源）
- 能量上限（限制单位时间产出）
- 多货币消耗（Travel Town的Coins+Tools双消耗）
- 物品有效期（事件专属物品过期消失）

## 四、变现设计

### 4.1 变现模式选型

| 模式 | 特征 | 适合方向 | 代表产品 |
|------|------|---------|---------|
| **纯IAP** | 体验纯净，依赖中高消费群体 | 高LTV产品 | Merge Mansion, Merge Dragons! |
| **混合IAP+IAA** | 广告补充收入，覆盖更广用户 | 大众休闲 | Travel Town, EverMerge |
| **广告主导** | IAP仅去广告，依赖广告量 | 超休闲 | Merge Town |

**选型决策**：
- 目标CPI高、追求高LTV → 纯IAP
- 目标用户规模大、需要广告收入补充 → 混合
- 超休闲定位、留存较短 → 广告主导

### 4.2 IAP定价策略

**付费金字塔**：

```
┌───────────┐
│ $49.99+   │ ← 鲸鱼包（大量硬货币+独占物品）
├───────────┤
│ $19.99    │ ← 高价值周期包
├───────────┤
│ $9.99     │ ← 中端事件包
├───────────┤
│ $4.99     │ ← 首充/新手包
├───────────┤
│ $1.99     │ ← 冲动购买入门包
└───────────┘
```

**付费层次设计**：

| 层次 | 价格 | 目标用户 | 设计要点 |
|------|------|---------|---------|
| 首充礼包 | $0.99-$4.99 | 首次付费用户 | 极高性价比，打破心理门槛 |
| 月卡/订阅 | $3.99-$9.99 | 持续付费用户 | 每日登录领取，30天绑定 |
| 事件包 | $4.99-$19.99 | 活跃付费用户 | 限时限量，紧迫感驱动 |
| 鲸鱼包 | $49.99-$99.99 | 高消费用户 | 大量资源+独占内容 |

### 4.3 激励视频最优触发点

| 触发场景 | 效果 | 频次建议 |
|---------|------|---------|
| 能量耗尽时 | 最高触发率 | 4-6次/天 |
| 资源选择翻倍 | 高采纳率 | 随机出现 |
| 离线回归翻倍 | 提升回访 | 每次回归1次 |
| 减少等待时间 | 中等采纳 | 不限 |
| 商店免费领取 | 稳定日活 | 每日1-2次 |

**激励视频 = IAP的试吃品**：广告奖励不应完全替代IAP价值，而是让玩家体验到付费的好处后产生购买欲。

### 4.4 脉冲式购买（Impulse Purchase）

EverMerge的经典设计：物品在玩家"差一个就能完成合成"时飘过屏幕。利用"差一点就完成"的心理紧迫感，付费购买后立即触发合成的满足感强化了正反馈。

### 4.5 Battle Pass / 订阅

- 解锁时机：5-7次会话后（面向已验证参与度的玩家）
- 双轨奖励：免费轨道保持非付费玩家活跃 + 付费轨道（$5-$10）
- 周期：通常30天
- 核心留存机制：未领取奖励产生损失厌恶

## 五、社交与留存设计

### 5.1 留存机制全景

```
┌─────────────────────────────────────────────────────────┐
│                      留存机制层次                         │
├──────────┬──────────────┬───────────────────────────────┤
│ 短期     │ 中期          │ 长期                          │
│ (每日)   │ (每周)        │ (每月+)                       │
├──────────┼──────────────┼───────────────────────────────┤
│ 每日签到  │ 周常任务      │ 收集系统（Album/图鉴）          │
│ Daily挑战 │ 里程碑事件    │ 赛季/Battle Pass              │
│ 能量恢复  │ 专属合成板    │ 叙事推进                      │
│ 限时礼盒  │ 装饰事件      │ 社交绑定                      │
│ 推送通知  │ 排行榜重置    │ 沉没成本积累                   │
└──────────┴──────────────┴───────────────────────────────┘
```

### 5.2 每日签到设计

```
递增奖励模型：
Day 1: 基础软货币
Day 2: 能量补充
Day 3: 合成材料
Day 4: 中级软货币
Day 5: 少量硬货币 ← 首次给出硬货币，关键留存节点
Day 6: 高级物品
Day 7: 大量硬货币/稀有宝箱 ← 周奖励高潮
```

**设计原则**：递增价值防止中途断签；断签暂停而非重置进度；允许硬货币补签。

### 5.3 限时活动矩阵

| 类型 | 周期 | 机制 | 目标 |
|------|------|------|------|
| 里程碑事件 | 3-4天 | 累计积分解锁阶梯奖励 | 中期参与 |
| 专属合成板 | 3-7天 | 独立限时合成地图 | 新鲜感 |
| 收集事件 | 持续 | 收集卡片/碎片 | 长期绑定 |
| 装饰事件 | 周期性 | 解锁限时装饰品 | 收集欲 |
| 竞速事件 | 1-3天 | 最快完成目标 | 短期爆发 |

### 5.4 社交系统（品类待开发领域）

当前Merge游戏品类的社交系统普遍薄弱。头部产品（EverMerge、Merge Mansion、Travel Town）均被行业分析指出社交系统是"最显著的缺失"。

**可行的社交设计方向**：
- 能量互赠（低开发成本，高留存价值）
- 合成板参观（社交比较+灵感来源）
- 合作事件（多人共同完成目标）
- 事件排行榜（短期竞争驱动变现）

### 5.5 回流与推送

**回流礼包梯度**：
- 1-3天离线：轻度欢迎（少量能量+软货币）
- 3-7天离线：中度挽回（大量能量+加速道具）
- 7-14天离线：重度挽回（硬货币+限时专属包）
- 14天+离线：重新引导（等同新用户激活流程）

**推送策略**：能量恢复满时、事件开始时、建造完成时、2-3天未登录时发送推送。

### 5.6 行业留存基准

头部Merge游戏的两个趋同指标：
- **D90/D1相对留存率**：约25%
- **日均使用时长**：约38分钟

## 六、核心方法论框架

### 框架一：三层满足感模型

```
┌────────────────────────────────────────┐
│ 第三层：完成感（Collection Satisfaction）│ ← 长期驱动
│   图鉴填充、区域解锁、角色收集           │
├────────────────────────────────────────┤
│ 第二层：发现感（Discovery Satisfaction） │ ← 中期驱动
│   新合成链发现、隐藏内容、意外产出        │
├────────────────────────────────────────┤
│ 第一层：合成感（Merge Satisfaction）     │ ← 短期驱动
│   即时合成反馈、连锁合成、视听反馈        │
└────────────────────────────────────────┘
```

### 框架二：摩擦-变现触发器矩阵

在自然摩擦点嵌入变现触发器：

| 摩擦点 | 玩家情绪 | 变现触发器 | 转化预期 |
|--------|---------|-----------|---------|
| 能量耗尽 | "还想再玩一会" | 能量包/广告 | 最高 |
| 差一个物品完成合成 | "就差一点" | 脉冲式购买 | 高 |
| 建造计时器 | "等不及" | 加速/跳过 | 中 |
| 库存空间满 | "放不下了" | 空间扩展 | 中 |
| 事件即将结束 | "来不及了" | 事件加速包 | 中高 |

### 框架三：产品定位四象限

```
        高策略深度
            │
   EverMerge │ Merge Dragons!
            │
  ──────────┼──────────
            │
 Travel Town│ Merge Mansion
            │
```

## 七、决策启发式

### 1. "30秒法则"
玩家从打开游戏到完成第一次合成不应超过30秒。首次合成的满足感决定了首日留存。

### 2. "永远差一个"原则
最佳的变现时机是玩家"差一个物品就能完成合成"的时刻。在这个时刻提供购买选项，转化率最高。

### 3. "三条活跃链"规则
任何时刻玩家应该有3-5条可推进的合成链。少于3条会感到无事可做，多于5条会感到混乱。

### 4. "确定性核心、随机性外围"
主线进度保持高确定性（玩家不会因运气差完全卡住），辅助奖励引入随机性（每次会话都有惊喜）。

### 5. "先宠溺再变现"
新手阶段给予远超稳态的资源量。让玩家体验"有钱"的快感后，自然进入资源稀缺阶段。

### 6. "能量是时间的货币化"
能量系统的本质是将玩家的时间转化为可售卖的商品。设计能量系统时，先定义目标会话频率和时长，再反推能量参数。

### 7. "社交是品类最大的未开发金矿"
当前头部Merge游戏的社交系统普遍薄弱。任何能在社交维度实现突破的产品都有机会获得结构性优势。

### 8. "脉冲比弹窗好"
脉冲式变现（物品自然飘过屏幕）比弹窗式Offer转化率更高，因为它融入了游戏流程而非打断游戏流程。

### 9. "Meta层决定天花板"
核心合成机制决定了产品的下限（是否好玩），Meta层（叙事/建造/收集）决定了产品的上限（能留住玩家多久）。

### 10. "通胀是沉默杀手"
资源产出增速必须略慢于消耗增速（约10%差距）。通胀一旦发生，回收过剩资源会引发玩家强烈不满，预防远优于治疗。
