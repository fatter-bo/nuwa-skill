# 研究维度三：经济系统

## 1. 软货币和硬货币的双轨制设计

### 双货币体系的基本结构

Merge游戏普遍采用双货币体系：

| 货币类型 | 获取方式 | 消耗场景 | 设计角色 |
|---------|---------|---------|---------|
| **软货币**（Coins/金币） | 合成产出、任务奖励、订单完成 | 购买基础物品、解锁区域、加速低级操作 | 进度推进润滑剂 |
| **硬货币**（Gems/Diamonds/宝石） | 少量游戏奖励、IAP购买 | 跳过计时器、购买稀有物品、解锁高级内容、购买能量 | 核心变现工具 |

### 各产品的货币策略

**Merge Mansion**：
- 硬货币（Gems）：新手赠送约100颗，随后急剧稀缺
- 设计哲学："先宠溺，再变现"（Spoil them, then monetize them）
- 软货币（Coins）相对充裕，不作为主要变现点

**Travel Town**：
- 硬货币（Diamonds）：核心变现载体
- 8000 Diamonds ≈ 20000能量（按标准兑换率）
- 软货币分为Coins和Tools双轨，约三分之一的阶段需要同时消耗两者

**EverMerge**：
- 硬货币（Rubies）：通过偶尔的游戏奖励和IAP获得
- 用于跳过建造计时器、解锁土地/英雄
- 经济系统通过"永远不够用的空间"制造自然稀缺

**来源**：Udonis Merge Mansion分析；Gamigion Travel Town拆解；Deconstructor of Fun EverMerge分析
**可信度**：高 — 多个专业平台交叉验证

## 2. 物品掉落概率表

### 生成器的概率产出设计

Merge游戏中的生成器（Generator）是物品的主要来源，其产出遵循概率表：

```
生成器产出概率表（示例结构）：
├── 1级物品：60-70%（最常见产出）
├── 2级物品：20-25%
├── 3级物品：5-10%
├── 稀有/特殊物品：1-3%
└── 空产出/废品：0-5%（部分产品使用）
```

### Travel Town的生成器类型

Travel Town在概率设计上引入了多种生成器变体：
- **无能量生成器**：冷却后自动在相邻格子产出物品
- **充能生成器**：需要5次点击激活，偶尔产出"缺陷变体"
- **事件专属生成器**：仅在限时活动中可用

### 概率设计原则

1. **保底机制**：连续N次低级产出后，提升高级产出概率
2. **新手保护期**：前几次使用生成器时提高高级物品概率
3. **内容节奏控制**：生成器概率随玩家等级动态调整，防止过快/过慢推进
4. **透明度平衡**：大多数Merge游戏不公开具体概率（不同于Gacha类游戏），保持发现感

**来源**：EverMerge分析指出"资源获取和能量奖励都有一定随机性，让游戏保持趣味"
**可信度**：中 — 概率数据多为社区反向工程，非官方公布

## 3. 资源产出与消耗的平衡模型

### Taps & Sinks（水龙头与水槽）框架

游戏经济的核心是Taps（产出源）和Sinks（消耗池）的平衡：

```
Taps（资源产出）          Sinks（资源消耗）
┌─────────────────┐      ┌─────────────────┐
│ 生成器点击产出    │      │ 合成消耗         │
│ 任务/订单奖励    │      │ 建造/解锁消耗    │
│ 事件奖励        │  ━━►  │ 商店购买         │
│ 每日登录奖励    │      │ 加速/跳过计时器   │
│ 广告观看奖励    │      │ 能量购买         │
│ IAP购买        │      │ 物品空间扩展      │
└─────────────────┘      └─────────────────┘
```

### 平衡设计的五步法

参考Game Developer的行业方法论：

1. **定义核心资源**：明确软硬货币、能量、物品的角色
2. **设计资源流向图**：画出所有产出源和消耗池
3. **设定产出速率**：基于目标会话时长（5-20分钟）反推每分钟的资源产出量
4. **设计消耗梯度**：随进度递增的消耗量，确保前期丰裕、中后期紧张
5. **动态调节**：通过A/B测试和数据分析持续微调

**来源**：Game Developer - "5 Basic Steps in Creating Balanced In-Game Economy"；Unity - "Building an in-game economy"；Machinations.io - "Game economy design in Free-to-Play games"
**可信度**：高 — 行业标准方法论，多个权威来源

### EverMerge的三系统平衡

EverMerge的经济设计要求玩家同时平衡三个子系统：
- **能量系统**：控制行动频率
- **建造计时器**：控制进度速度（越高级计时越长）
- **库存空间**：控制物品累积上限

三系统的交互创造了持续的决策空间："玩家必须不断平衡这三个输入与可用的游戏板空间"。

**来源**：Deconstructor of Fun - EverMerge分析
**可信度**：高

## 4. 通胀控制

### 通胀的表现形式

在Merge游戏中，"通胀"表现为：
- 软货币过度累积导致购买力过剩
- 物品产出速度超过消耗速度导致库存溢出
- 高级物品过早获得导致后续内容失去吸引力

### 反通胀机制

| 机制 | 原理 | 产品示例 |
|------|------|---------|
| **空间限制** | 固定的游戏板面积限制物品总量 | 所有Merge游戏 |
| **递增消耗** | 后续内容需要更多资源 | EverMerge建造计时器递增 |
| **能量上限** | 限制单位时间内的资源产出 | 全行业标准 |
| **物品有效期** | 限时物品过期消失 | 事件专属物品 |
| **等级门槛** | 高级消耗选项随等级解锁 | Travel Town工具系统 |
| **多货币消耗** | 同时需要多种资源 | Travel Town的Coins+Tools双消耗 |

### 核心反通胀公式

```
理想状态：资源产出增速 < 资源消耗增速 × 1.1

即：随着玩家进度推进，消耗的增长速度应略快于产出的增长速度，
维持约10%的"稀缺压力"，既不至于完全卡死，又保持付费动力。
```

**来源**：Game Developer经济设计方法论；Unity经济平衡指南；行业实践总结
**可信度**：中高 — 基于行业通用原则，非特定产品的精确数据
