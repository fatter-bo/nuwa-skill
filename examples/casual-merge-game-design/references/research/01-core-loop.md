# 研究维度一：核心循环设计

## 1. Merge游戏的基础循环

### 发现-合成-升级-收集 四阶循环

Merge游戏的核心循环可以抽象为四个阶段的闭环：

```
发现(Discover) → 合成(Merge) → 升级(Upgrade) → 收集(Collect)
       ↑                                              |
       └──────────────────────────────────────────────┘
```

- **发现**：通过能量消耗、生成器产出、任务奖励等方式获得新物品
- **合成**：将相同物品合并为更高等级物品（Merge-2或Merge-3机制）
- **升级**：合成后获得更高级物品，解锁新的合成路径
- **收集**：完成特定物品链、填充图鉴、达成任务目标

**来源**：Deconstructor of Fun - "How EverMerge Made $50M in just 7 Months"（2020）；GameRefinery - "Why Merge Could be the New Match3"
**可信度**：高 — 行业权威分析平台，基于实际产品数据

### Merge-2 vs Merge-3 机制差异

| 维度 | Merge-2（合二为一） | Merge-3（合三为一） |
|------|-------------------|-------------------|
| 代表产品 | Merge Mansion, Travel Town | Merge Dragons!, EverMerge |
| 操作门槛 | 更低，两个即可合成 | 稍高，需凑齐三个 |
| 策略深度 | 偏低，决策简单 | 较高，存在3合vs5合的策略选择 |
| 空间压力 | 更大（物品更快堆积） | 较小（3→1压缩率更高） |
| 满足感节奏 | 频繁的小满足 | 间隔的大满足 |

**来源**：Udonis - "Merge Mansion Monetization Strategy"；Gamigion - "Travel Town Deconstruction"
**可信度**：高 — 基于头部产品实际拆解

## 2. 合成树（Merge Tree）的深度与宽度设计

### 深度设计原则

合成树的"深度"指从最低级物品到最终物品的合成层数。

- Merge Dragons的物品链平均深度在8-12级
- EverMerge的建筑链深度在5-8级
- Travel Town的物品链深度在6-10级

**设计规律**：
1. 前3级应该在单次会话内可达成（给新手即时满足感）
2. 中间层级（4-7级）构成日常游戏的主要内容
3. 最后2-3级需要多日积累才能完成（长期目标锚定）

### 宽度设计原则

合成树的"宽度"指同时存在的平行合成链数量。

- EverMerge同时维持多条资源链（木材、石头、金矿等），各链有不同的合成机制
- Travel Town采用专项链（nestled chains）、组合链（combined chains）等多种链类型增加宽度

**宽度与深度的平衡**：
- 宽度过大 → 玩家注意力分散，库存压力大
- 宽度过小 → 内容单调，容易疲劳
- 行业最佳实践：同时活跃的合成链控制在3-5条

**来源**：Funovus - "Merge Dragons: Finding the Right Slice"；Deconstructor of Fun EverMerge分析
**可信度**：高 — 开发者第一方分享 + 行业权威拆解

## 3. 合成链的满足感曲线

### 5-Merge奖励机制

在Merge-3游戏中，玩家可以选择3个合成（获得1个高级物品）或5个合成（获得2个高级物品）。EverMerge的分析显示，这个"20%进度提升"创造了一个"轻量但极其令人满足的掌握元素"。

### 满足感曲线的设计要点

```
满足感
  ↑
  │        ★ 首次完成一条完整合成链
  │    ★        
  │  ★   ★         ★ 发现隐藏合成路径
  │ ★      ★     ★
  │★         ★ ★       （低谷：中间层级的磨时间阶段）
  │
  └──────────────────────────→ 游戏进度
```

**关键设计原则**：
1. **首次合成即高潮**：新玩家的第一次合成必须产生强烈的正反馈（动画、音效、视觉升级）
2. **链式反应奖励**：当合成结果恰好落在相同物品旁触发连锁合成时，给予额外奖励（Merge Dragons的Combo机制）
3. **里程碑奖励**：每完成一条完整合成链给予特殊奖励，对冲中间层级的磨损感
4. **意外发现**：偶尔在合成中产出意外物品，维持新鲜感

**来源**：Deconstructor of Fun；Matt Bernstein - "Merge Dragons: A Critical Play"（Medium）
**可信度**：中高 — 分析文章基于深度体验，非官方数据

## 4. "发现"机制的随机性与确定性平衡

### Merge Dragons的发现哲学

Merge Dragons的设计哲学核心之一是"发现"：游戏刻意不向玩家展示完整的合成链和最终产物。玩家只能看到已遇到过的物品描述，未知的部分始终保持神秘。这种设计让发现成为自然的游戏进程。

**GDC分享要点（GDC Vault - "Design Philosophies and Lessons from Developing Merge Dragons!"）**：
- 不揭示"路的尽头"，让玩家始终好奇前方还有什么
- 发现、无尽游戏、"合成一切"是三大设计支柱

### 随机性与确定性的配比

| 机制 | 确定性 | 随机性 | 设计意图 |
|------|--------|--------|----------|
| 合成结果 | 高（固定产出） | 低 | 给玩家可预期的进度感 |
| 生成器产出 | 中 | 中（概率表） | 增加每次点击的期待感 |
| 任务奖励 | 高 | 低 | 保证进度可控 |
| 事件掉落 | 低 | 高 | 制造惊喜和稀缺感 |
| 宝箱内容 | 低 | 高 | 变现触发点 |

**设计原则**：
- 核心进度路径保持高确定性（玩家不会因为运气差而完全停滞）
- 辅助奖励路径引入随机性（保持每次会话的新鲜感）
- 随机性的"底线保护"：即使运气最差，生成器也应保证最低产出

**来源**：GDC Vault - Merge Dragons设计分享；Funovus开发者博客
**可信度**：高 — GDC官方演讲，开发者第一方内容
