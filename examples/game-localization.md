# 游戏本地化专家 · 全维度方法论系统

> 本地化不是翻译。翻译是把中文变成日语；本地化是把一款中国游戏"变成"一款日本游戏。
> 真正的本地化覆盖语言、文化、数值、付费、合规、工作流六个维度。

## 一、文化敏感性框架

### 框架1：颜色-数字-符号三层过滤器

对每个目标市场，逐层检查：

**第一层：颜色**
| 颜色 | 中国 | 日本 | 韩国 | 中东 | 西方 |
|------|------|------|------|------|------|
| 红 | 喜庆/好运 | 生命力/危险 | 热情 | 危险 | 爱/危险 |
| 白 | 丧葬 | 丧葬/纯洁 | 丧葬 | 纯洁 | 纯洁/婚礼 |
| 绿 | "绿帽"=负面 | 永恒 | 青春 | 神圣(伊斯兰) | 自然/金钱 |
| 黑 | 不祥 | 正式/丧葬 | 权威 | 重生/哀悼 | 高雅/死亡 |

**第二层：数字**
- 4 → 中日韩忌讳（谐音"死"）：跳过4号楼层/关卡/定价
- 13 → 西方忌讳：避免13层/13号
- 8 → 中国大吉：定价尾数用8
- 666 → 西方=恶魔数字 / 中国=口语"溜溜溜"

**第三层：宗教与符号**
- 伊斯兰：禁止先知形象描绘、古兰经文字装饰化、猪形象需替换
- 佛教（泰国）：佛像不可作为踩踏/负面元素
- 旭日旗：日本常见 → 韩国/中国视为军国主义象征，必须移除
- 十字架：基督教文化中作为负面符号可能引发争议

## 二、UI/UX本地化框架

### 框架3：文本膨胀预算系统

从英语出发的膨胀率：
- **德语**：+30%~40%（最长，按钮/标签设计基准）
- **法/西/俄**：+20%~35%
- **阿拉伯语**：+25%~30% + RTL全面适配
- **日/中/韩**：-20%~+10%（更紧凑，但需更大字号）
- **泰语**：+15%~20% + 额外行高（声调标记）

**设计黄金法则**：所有UI元素从一开始预留40%文本扩展空间。

### RTL适配清单

**必须镜像**：导航栏、返回/前进箭头、进度条、列表布局、滑动方向、文本对齐

**不需镜像**：时钟图标、播放/暂停、对勾/叉号、数字方向、品牌logo

### 字体Fallback链策略
```
主字体 → 语言专用字体 → Google Noto Sans [语言] → 系统默认字体
```

## 四、付费模型本地化框架

### 框架5：PPP定价公式

```
本地价格 = 基准价格(USD) * 汇率 * PPP调整系数 * 心理价格圆整
```

**心理定价规则**：
- 美国：$X.99尾（$4.99、$9.99）
- 日本：整数（610円、3,060円）
- 中国：吉利数尾（68元、88元、168元、648元）
- 东南亚：超低起步（$0.10-$1.00）

**本地化定价收入提升：20%-40%**（行业统计）

## 五、法规合规框架

### 框架6：合规决策树

```
目标市场确定
  ├── 是否收集用户数据？
  │   ├── EU → GDPR (16岁以下需父母同意, 罚款最高全球营收4%)
  │   ├── US → COPPA (13岁以下, 2026.04新规生效)
  │   ├── 韩国 → PIPA (实名认证+概率公示)
  │   ├── 中国 → PIPL (数据本地化+防沉迷)
  │   └── 日本 → APPI (跨境传输限制)
  │
  ├── 是否有随机付费内容(loot box/gacha)？
  │   ├── 比利时 → 完全禁止
  │   ├── 荷兰 → 可交易物品禁止
  │   ├── 中国 → 概率公示+渐进提升+每日限购
  │   ├── 日本 → Kompu Gacha禁止, JOGA自律公示
  │   ├── 韩国 → GRAC强制概率公示
  │   ├── 澳大利亚 → 强制M分级(15+)
  │   └── 美国 → FTC执法趋严($2000万原神案)
  │
  └── 需要哪些年龄分级？
      ├── 北美 → ESRB
      ├── 欧洲 → PEGI
      ├── 日本 → CERO
      ├── 韩国 → GRAC
      ├── 台湾 → GSRR
      ├── 德国 → USK
      ├── 澳大利亚 → ACB
      └── 数字平台 → IARC (一次问卷映射多地区)
```

## 启发式规则（Heuristics）

1. **40%规则**：所有UI元素从设计之初就预留40%的文本扩展空间。德语是你的压力测试语言。

2. **颜色不是通用语言**：红色在中国=喜庆，在中东=危险。绿色在中国="绿帽"=侮辱，在伊斯兰=神圣。永远查目标市场的颜色语义表。

3. **法规是硬边界，文化是软边界**：比利时禁止loot box就是不能上，没有商量余地。文化适配则有灰度——需要本地专家判断。

4. **定价本地化的20-40%法则**：根据购买力平价调整IAP定价，收入可提升20%-40%。$4.99在美国合理，在印度可能需要降到$0.50。

5. **支付方式决定转化率**：在东南亚不接入运营商代扣，在印度不接入UPI，等于放弃了大量潜在付费用户。

6. **翻译 ≠ 本地化 ≠ 文化化**：翻译改文字，本地化改格式和习惯，文化化改内容和体验。三个层次缺一不可。

7. **Pseudo-localization是最便宜的保险**：在开发阶段用伪本地化（用带重音字符替换文本）检测UI硬编码和截断问题，成本几乎为零。

8. **日本市场是"另一个世界"**：ARPU是全球4倍，接受无保底gacha，偏好精致短session。不要用中国或欧美的框架套用。

9. **IARC一次填表多地区分级**：数字平台(Google Play / Microsoft Store)使用IARC系统，一份问卷自动映射到ESRB/PEGI/USK等多个分级。Sony PlayStation仍需单独申请。

10. **AI翻译做初稿，人工做灵魂**：2026年的最佳实践是AI处理标准UI文本，人工专家聚焦叙事/对话/文化敏感内容。可降低30-50%成本。

## 诚实边界

1. **法规信息时效性**：各国游戏法规快速演变，本Skill中的法规信息基于2026年4月调研，使用前应确认最新政策
2. **ARPU等数据为近似值**：不同来源的市场数据存在差异，具体数字应参考Sensor Tower/data.ai等专业平台
3. **文化判断需本地专家验证**：文化敏感性存在灰度地带，关键决策应咨询目标市场的本地化专家或文化顾问
4. **AI翻译质量因语言而异**：AI在欧洲语言翻译上质量较好，对小语种/文学性文本仍需更多人工介入
5. **定价策略需A/B测试**：PPP定价公式是起点而非终点，最终定价应通过实际市场A/B测试确定
