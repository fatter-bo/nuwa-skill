# 研究维度三：留存诊断框架

## 1. 留存曲线形态分析

### 三种典型留存曲线

**形态一：自然衰减型（Healthy Decay）**
- 特征：D1-D3快速下降，D7-D14放缓，D20+趋于平坦
- 含义：产品健康，用户自然筛选后留下核心群体
- 关键观察点：D20-D25通常出现拐点，此后留存曲线趋平，D30留存可较好预测D365留存
- 对策：正常优化，关注各阶段留存绝对值是否达标

**形态二：断崖型（Cliff Drop）**
- 特征：某一天或某个阶段出现急剧下降（如D1到D2掉50%+）
- 含义：存在明确的体验断裂点——内容耗尽、难度骤增、强制付费墙、Bug
- 典型场景：D1-D2断崖（新手引导后无事可做）、D3-D5断崖（内容消耗完毕）、D7断崖（周活动结束/试用期到期）
- 对策：定位断崖发生的具体天数，关联游戏内容/功能解锁时间表

**形态三：回升型（Recovery/Bounce）**
- 特征：在某个时间点后留存出现回升
- 含义：通常由外部因素驱动——推送通知、限时活动、版本更新、营销召回
- 注意：区分真实产品改善 vs 一次性营销刺激
- 对策：分析回升是否可持续，如果依赖外部刺激则需建立常态化机制

**来源**：Solsten "The True Drivers Of D1, D7, And D30 Retention In Gaming"; GameAnalytics 2025 Benchmarks
**可信度**：高 — 行业研究+大样本统计数据

## 2. D1留存低的常见原因与排查顺序

D1留存反映的是"第一印象"——玩家是否在首次体验中发现了乐趣。

### 排查清单（按优先级）

**P0：Fun Discovery失败（最高优先级）**
- 症状：D1 < 25%且Session时长极短（<2分钟）
- 原因：核心玩法在30秒内未被触达，玩家不知道"这个游戏好玩在哪里"
- 排查：检查首次Session的漏斗——从打开App到完成第一个有正反馈的操作需要多久
- 行动：将"Aha Moment"前移到30秒以内

**P0：新手引导设计失败**
- 症状：D1 < 30%，tutorial_completed率低，tutorial_skipped率高
- 原因：教程过长、文字过多、缺少互动、未展示核心乐趣
- 排查：检查教程每步的完成率漏斗，找到最大流失步骤
- 行动：渐进式教学（Learn by Doing），缩短到3-5步，每步有即时正反馈

**P1：技术障碍**
- 症状：D1低且崩溃率>1%或加载时间>5秒
- 原因：App崩溃、加载过慢、低端设备不兼容、网络要求过高
- 排查：按设备型号/OS版本分组对比D1留存，检查首Session崩溃率
- 行动：性能优化，加载时间<3秒，崩溃率<0.5%

**P1：受众不匹配**
- 症状：高量级渠道的D1留存显著低于自然量
- 原因：广告素材传达的预期与实际游戏体验不符
- 排查：按渠道/Campaign/创意素材分组对比D1留存
- 行动：让广告素材真实反映核心玩法，而非吸引眼球但与游戏无关的内容

**P2：首日目标缺失**
- 症状：D1在28-35%徘徊，难以突破
- 原因：玩家完成教程后缺少明确的首日目标
- 行动：设计"首日任务链"，给玩家明确的D1目标和完成奖励

**来源**：Solsten D1/D7/D30 Retention Framework; Mistplay Retention Benchmarks; Playio DAU Retention Analysis
**可信度**：高 — 综合多个行业研究

## 3. D7留存低的常见原因与排查

D7留存反映"习惯养成"——用户是否将游戏融入日常。

### 排查清单

**P0：内容耗尽**
- 症状：D1正常但D3-D7急剧下降
- 原因：7天内可体验的内容被快速消耗完毕
- 排查：检查D1-D7每天的Session数和时长变化趋势
- 行动：确保至少2周的内容深度，引入程序化生成/随机性

**P0：进阶系统缺乏吸引力**
- 症状：D7低但D1正常，核心循环参与度下降
- 原因：解锁新手期后，进阶玩法/Meta层不够有趣
- 排查：检查D3-D7期间新功能解锁后的使用率
- 行动：在D2-D5持续解锁新机制，保持新鲜感

**P1：社交绑定不足**
- 症状：有好友的用户D7留存显著高于无好友的
- 原因：游戏缺少社交连接点
- 排查：按"是否添加好友"分群对比D7留存
- 行动：在D1-D3引导添加好友/加入公会，好友互动机制

**P1：磨损感过重（Grind Wall）**
- 症状：进度曲线在D3-D5变平，Session时长下降
- 原因：进度回报感知变弱，"打磨"感觉太重
- 排查：检查每日进度推进量的变化趋势
- 行动：调整难度曲线，增加"小胜利"频率

**P2：每日激励不足**
- 症状：D7间歇性回访但非每日
- 原因：缺少每日回来的理由
- 行动：递增签到奖励、每日任务、限时活动

## 4. D30留存低的常见原因与排查

D30留存反映"长期忠诚"——用户是否对游戏产生了真正的投入感。

### 排查清单

**P0：Meta层深度不足**
- 症状：D7尚可但D14-D30持续衰减
- 原因：缺少叙事/建造/收集等长期目标
- 排查：D14+用户的功能使用分布，是否还在探索新内容
- 行动：引入Battle Pass、赛季系统、收集系统、叙事线

**P0：LiveOps节奏缺失**
- 症状：无活动期间D30下降，有活动时回升
- 原因：缺乏持续的新鲜内容刺激
- 排查：对比活动期与非活动期的留存曲线
- 行动：建立周期性活动日历，每周至少1个限时事件

**P1：社交流失连锁**
- 症状：好友/公会活跃度下降后，个人留存跟随下降
- 原因：社交网络瓦解导致游戏动力丧失
- 排查：检查流失用户的好友活跃度变化
- 行动：活跃好友推荐、跨服社交、好友召回机制

**P1：经济系统失衡**
- 症状：付费用户留存远高于免费用户
- 原因：非付费用户体验受到过度压制
- 排查：按付费/非付费分群对比D30留存和进度
- 行动：确保F2P用户有合理的进度路径

## 5. 留存诊断决策树

```
D1 < 25%?
  ├─ Yes → 检查FTUE（首30秒体验）→ 检查崩溃率/加载时间 → 检查渠道匹配度
  └─ No → D1 25-35%?
          ├─ Yes → 优化教程设计 → 设计首日目标链 → A/B测试首日体验
          └─ No（D1 > 35%）→ 检查D7
                              D7 < 10%?
                              ├─ Yes → 检查内容深度 → 检查进阶系统 → 检查社交绑定
                              └─ No → D7 > 10% → 检查D30
                                      D30 < 5%?
                                      ├─ Yes → 检查Meta层深度 → 检查LiveOps → 检查经济平衡
                                      └─ No → 产品留存健康，关注变现与增长
```

## 6. 留存分析的进阶方法

### 按维度拆分分析

- **按渠道**：自然量 vs 付费量 vs 社交量，诊断获客质量
- **按地区**：不同国家的留存差异可能指向本地化问题
- **按设备**：低端设备留存低可能是性能问题
- **按版本**：版本前后留存变化定位功能影响
- **按用户行为**：将"D1内完成X"作为分组条件，识别留存驱动行为

### Cohort Analysis（同期群分析）

按安装日期分组，追踪每个Cohort的留存曲线，发现：
- 哪些版本改善/恶化了留存
- 哪些Campaign带来了高/低质量用户
- 季节性因素的影响

**来源**：Playio "DAU and Retention Analysis for Mobile Games 2026"; Solsten Retention Framework; GameAnalytics Benchmarks
**可信度**：高 — 结合行业框架与一手数据分析方法论
