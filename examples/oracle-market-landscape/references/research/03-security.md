# 03-security: 预言机安全模型与攻击向量

## 历史安全事件复盘

### bZx闪电贷攻击（2020.02）
- **损失**：~$95.4万
- **攻击流程**：通过闪电贷借$1000万ETH → 操纵依赖单一预言机的抵押品池价格 → 套利
- **根因**：依赖单一预言机确定价格
- **教训**：必须使用多源聚合预言机

### Harvest Finance（2020.10）
- **损失**：$3400万
- **攻击流程**：闪电贷操纵Curve池价格 → 该价格被用作预言机
- **根因**：使用DEX AMM价格作为预言机来源
- **教训**：DEX价格不应直接用作预言机——可被闪电贷操纵

### Warp Finance（2020.12）
- **损失**：$770万
- **攻击流程**：操纵LP token价格
- **根因**：LP token定价逻辑可被操纵

### Cream Finance（2021.10）
- **损失**：$1.3亿
- **攻击流程**：预言机操纵
- **根因**：预言机定价逻辑被利用

### Mango Markets（2022.10）
- **损失**：$1.17亿
- **攻击流程**：用$1000万USDC跨两个账户人为拉高MNGO价格 → 用虚高token做抵押借$1.17亿其他资产
- **根因**：低流动性token + 链上TWAP预言机可被操纵
- **教训**：TWAP在低流动性环境中不安全

### UMA/Polymarket鲸鱼操纵（2025.03-12）
- **事件1**：乌克兰矿产市场$700万——一个鲸鱼用500万UMA代币（25%投票权）操纵结果
- **事件2**：UFO文件市场$1600万——类似的鲸鱼投票操纵
- **事件3**：Zelenskyy穿西装市场$2.15亿对——尽管有照片证据仍被投票为"No"
- **根因**：代币集中度过高 + 投票参与率过低 + CoC < PfC
- **教训**：代币加权投票在高价值场景中不安全

### AWS宕机事件（2025.10.19-20）
- US-EAST-1区域15小时宕机
- Chainlink：正常运行（多云分布）
- LayerZero：宕机
- Circle CCTP：宕机迹象
- **教训**：运营去中心化（多云多区域）是必需的

## 攻击向量分析

### 1. 闪电贷操纵
- **原理**：借大量资金→操纵低流动性价格→利用被操纵的价格套利→归还贷款——全在一个交易中
- **防御**：使用TWAP（但TWAP也可被操纵）、使用多源聚合、设置价格偏差阈值

### 2. 数据源攻击
- **原理**：攻击喂入预言机节点的API端点、操纵低流动性交易所的价格
- **防御**：多数据源聚合、异常值过滤、数据源信誉系统

### 3. MEV抢跑
- **原理**：在mempool中看到预言机更新交易→在更新前后夹持交易（三明治攻击）
- **规模**：以太坊2025年$12亿+ MEV提取，三明治攻击占51%
- **防御**：私有mempool（Flashbots）、MEV返还机制（API3 OEV, UMA Oval）

### 4. 私钥泄露
- **原理**：单签名预言机——一个密钥被攻破即完全被控
- **防御**：多签（N-of-M）、硬件安全模块（HSM）、密钥轮换、地理分布

### 5. 重放攻击
- **原理**：将一条链上的签名预言机数据重放到另一条链
- **防御**：签名数据包含chainID和时间戳、nonce/序列号

### 6. 投票操纵（乐观预言机专属）
- **原理**：集中代币投票权强行通过错误结果
- **条件**：代币集中度高 + 投票参与率低 + 市场价值 > 操纵成本
- **防御**：提高法定人数、多代币投票、白名单提案者（但牺牲去中心化）

## 中心化预言机信任构建清单

| 要素 | 实现方式 | 验证方法 |
|------|---------|---------|
| 代码透明 | 开源所有代码（GitHub） | 任何人可审计 |
| 密钥安全 | 多签（3-of-5+）、HSM、地理分布 | 链上多签地址可验证 |
| 可审计性 | 每次更新都是链上交易 | 区块浏览器可查 |
| SLA | 合同约定99.9%+可用性和准确性 | 监控仪表板 |
| 声誉 | 历史表现记录（准确率、响应时间） | 链上统计 |
| 冗余 | 多云多区域（AWS+GCP+Azure） | 基础设施文档 |
| 经济激励 | 质押/罚没机制 | 智能合约逻辑 |
